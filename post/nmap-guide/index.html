Nmap — Field Guide

2026-06-02 · 1 min read · Tools · nmap recon scanning network tools

Scan types, essential flags, NSE scripts, and ready-to-run examples for network discovery, service enumeration, and CTF recon.

01 What is Nmap

Nmap (Network Mapper) is the industry-standard tool for host discovery, port scanning, service fingerprinting, OS detection, and scripted vulnerability checks. It's the first tool you run on a new target — whether that's an internal network audit, a bug bounty scope, or a HackTheBox machine.

Basic syntax:

text 
nmap [scan type] [options] <target>

Target can be an IP (10.10.10.5), a range (10.10.10.0/24), a list (-iL hosts.txt), or a hostname.

02 Scan Types

FlagNameNotes
`-sS`SYN stealthDefault when root. Sends SYN, never completes handshake. Fast, less logged.
`-sT`TCP connectFull handshake. Used without root. More detectable.
`-sU`UDPSlow but essential — DNS (53), SNMP (161), NFS (2049) live here.
`-sA`ACKMaps firewall rules, not open ports.
`-sN`NULLNo flags set. Bypasses some firewalls.
`-sF`FINFIN flag only. Same bypass use-case as NULL.
`-sX`XmasFIN+PSH+URG. Lights up like a Christmas tree.
`-sV`VersionProbe open ports for service + version info.
`-sC`Default scriptsRuns NSE scripts in the `default` category.
bash Scan type examples 
# SYN scan (requires root/sudo)
sudo nmap -sS 10.10.10.5

# TCP connect scan (no root needed)
nmap -sT 10.10.10.5

# UDP scan — slow, run on top ports only
sudo nmap -sU --top-ports 100 10.10.10.5

# Combine TCP + UDP
sudo nmap -sS -sU 10.10.10.5

# ACK scan to check firewall filtering
sudo nmap -sA 10.10.10.5

03 Port Selection

bash Specifying ports 
# Single port
nmap -p 80 10.10.10.5

# Range
nmap -p 1-1000 10.10.10.5

# Comma-separated list
nmap -p 22,80,443,8080,8443 10.10.10.5

# All 65535 ports (thorough but slow)
nmap -p- 10.10.10.5

# Top N most common ports
nmap --top-ports 1000 10.10.10.5

# Fast scan — top 100 ports only
nmap -F 10.10.10.5

# Only show open ports (suppress filtered/closed noise)
nmap --open 10.10.10.5

04 Timing & Performance

Nmap's -T flag controls the timing template — a balance between speed and stealth/reliability.

TemplateNameUse case
`-T0`ParanoidIDS evasion. One probe every 5 minutes.
`-T1`SneakySlow, low noise. Still painful.
`-T2`PoliteReduces bandwidth. Unreliable networks.
`-T3`NormalDefault.
`-T4`AggressiveFast. Good for reliable networks / CTF.
`-T5`InsaneVery fast, may miss results.

For HTB/CTF: use -T4. For real-world pentests on production: stick to -T3 or lower.

bash Timing examples 
# Fast recon on a CTF box
sudo nmap -T4 -p- 10.10.10.5

# Slow stealth scan to avoid IDS alerts
sudo nmap -T1 -sS 192.168.1.0/24

# Parallel host probing (--min-parallelism speeds up subnet scans)
sudo nmap -T4 --min-parallelism 50 192.168.1.0/24

05 Host Discovery

Before scanning ports, Nmap pings hosts to check if they're alive. These flags control that behaviour.

bash Host discovery flags 
# Ping sweep — find live hosts on a subnet (no port scan)
sudo nmap -sn 192.168.1.0/24

# Skip ping check — treat host as up (useful when ICMP is blocked)
sudo nmap -Pn 10.10.10.5

# No DNS resolution (faster, avoids DNS logging)
nmap -n 10.10.10.5

# Scan from a list of targets
nmap -iL targets.txt

# Exclude specific hosts from a range
nmap 192.168.1.0/24 --exclude 192.168.1.1,192.168.1.254

06 Service & OS Detection

bash Detection flags 
# Service version detection
nmap -sV 10.10.10.5

# OS detection (requires root)
sudo nmap -O 10.10.10.5

# Aggressive mode: -sV + -O + -sC + traceroute
sudo nmap -A 10.10.10.5

# Crank up version probe intensity (0-9, default 7)
nmap -sV --version-intensity 9 10.10.10.5

# Verbose output — see what Nmap is doing live
nmap -v -sV 10.10.10.5
nmap -vv -sV 10.10.10.5  # even more verbose

07 Nmap Script Engine (NSE)

NSE scripts extend Nmap's capabilities with service enumeration, vulnerability detection, brute-force, and more. Scripts live in /usr/share/nmap/scripts/.

Script categories:

CategoryWhat it does
`default`Safe, useful — runs with `-sC`
`discovery`Enumerates services deeper
`safe`No destructive actions
`vuln`Checks for known CVEs
`auth`Tests for auth bypass / weak creds
`brute`Password brute-forcing
`exploit`Actual exploitation (use carefully)
bash NSE examples 
# Run default scripts (same as -sC)
nmap --script default 10.10.10.5

# Vulnerability scan
sudo nmap --script vuln 10.10.10.5

# Check for EternalBlue (MS17-010)
nmap --script smb-vuln-ms17-010 -p 445 10.10.10.5

# HTTP enumeration
nmap --script http-title,http-headers,http-methods -p 80,443,8080 10.10.10.5

# FTP anonymous login check
nmap --script ftp-anon -p 21 10.10.10.5

# SSH supported auth methods
nmap --script ssh-auth-methods -p 22 10.10.10.5

# SMB enumeration
nmap --script smb-enum-shares,smb-enum-users -p 445 10.10.10.5

# SSL/TLS certificate info
nmap --script ssl-cert,ssl-enum-ciphers -p 443 10.10.10.5

# Run all scripts in a category
sudo nmap --script discovery 10.10.10.5

# Pass arguments to a script
nmap --script http-brute --script-args userdb=users.txt,passdb=pass.txt -p 80 10.10.10.5

08 Output & Saving Results

bash Output formats 
# Normal text output
nmap -oN scan.txt 10.10.10.5

# XML output (importable into Metasploit, etc.)
nmap -oX scan.xml 10.10.10.5

# Grepable output
nmap -oG scan.gnmap 10.10.10.5

# All three at once (recommended — saves everything)
nmap -oA scan 10.10.10.5
# Produces: scan.nmap, scan.xml, scan.gnmap

# Resume an interrupted scan
nmap --resume scan.gnmap

09 Real-World Use Cases

Network asset inventory — discover what's running across a subnet before an audit.

bash Network inventory scan 
# Ping sweep to find live hosts first
sudo nmap -sn 10.0.0.0/24 -oG alive.gnmap
grep 'Up' alive.gnmap | awk '{print $2}' > live_hosts.txt

# Full version + script scan on live hosts
sudo nmap -sS -sV -sC -T3 -p- -oA full_audit -iL live_hosts.txt

Firewall validation — confirm that only intended ports are reachable from outside.

bash Firewall rule check 
# Check which ports respond vs which are filtered
sudo nmap -sA -p 22,80,443,3306,5432 10.0.0.1

# Test firewall bypass with fragmented packets
sudo nmap -f -sS 10.0.0.1

# Decoy scan to obscure source IP (mix real IP with decoys)
sudo nmap -D 192.168.1.50,192.168.1.51,ME 10.0.0.1

Scheduled monitoring — lightweight recurring scan to detect unexpected new open ports.

bash Minimal monitoring scan 
# Quick top-100-ports scan, grepable output, no DNS
sudo nmap -n -T3 --top-ports 100 --open -oG - 10.0.0.0/24 \
  | grep 'Ports:' >> port_history.log

10 CTF & HackTheBox Workflow

On CTF boxes, the goal is speed and completeness. A two-phase approach works well: fast scan first to find low-hanging fruit, then a thorough scan in the background.

bash HTB / CTF recon workflow 
export IP=10.10.10.5

# Phase 1 — fast top-port scan while you read the box description
sudo nmap -sS -T4 --top-ports 1000 --open -oN fast_$IP.txt $IP

# Phase 2 — full port scan in background
sudo nmap -sS -T4 -p- --open -oN allports_$IP.txt $IP &

# Phase 3 — version + default scripts on discovered ports
# (replace 22,80,443 with what Phase 1 found)
sudo nmap -sV -sC -p 22,80,443 -oN services_$IP.txt $IP

# Phase 4 — UDP top ports (often reveals SNMP, NFS, etc.)
sudo nmap -sU --top-ports 50 -oN udp_$IP.txt $IP
bash Useful one-liners for CTF 
# All-in-one aggressive scan (good first scan on a known-safe HTB box)
sudo nmap -A -T4 -p- --open $IP

# Check for common CVEs quickly
sudo nmap --script vuln -p $(cat allports_$IP.txt | grep open | cut -d/ -f1 | tr '\n' ',') $IP

# SMB deep dive
nmap --script smb-vuln-ms17-010,smb-enum-shares,smb-os-discovery -p 445 $IP

# Web enumeration combo
nmap --script http-title,http-auth-finder,http-methods,http-robots.txt -p 80,443,8080,8443 $IP
Cheat card

Always save output with -oA scanname — you'll want to grep it later.
Skip ping with -Pn when a host is alive but not responding to ICMP.
Add -n to skip DNS lookups on internal/CTF scans — noticeably faster.
-sV slows things down — run it only on ports you've already confirmed open.
Root/sudo is required for SYN (-sS), UDP (-sU), and OS detection (-O).
