Q2 2026 Threat Landscape – Attackers Target Trust Infrastructure

2026-06-03 · 3 min read · News · cve zero-day active-exploitation

Attackers in Q2 2026 are bypassing endpoints entirely and going straight for identity systems, VPNs, device managers, and security tools. A breakdown of actively exploited CVEs, macro trends, and what defenders should prioritise now.

01 The Theme: Hit the Trust Layer, Not the Endpoint

The defining pattern of Q2 2026 is not novel malware or exotic techniques — it is target selection. Threat actors are consistently choosing systems that other systems inherently trust: VPN gateways, identity providers, MDM platforms, collaboration hubs, and the security products meant to detect them.

Compromise one of these and you do not need a foothold on every workstation. You have the keys to the domain.

Attack Chain PhasePreferred Target Class
Initial AccessVPN / remote-access gateways
Auth BypassIdentity providers, SSO, MFA infrastructure
Privilege EscalationSecurity products (EDR/AV)
Lateral MovementCollaboration platforms (SharePoint, Teams)
PersistenceMDM / device management platforms
Defense EvasionSecurity controls themselves

02 Actively Exploited CVEs

The following vulnerabilities have confirmed in-the-wild exploitation as of Q2 2026. Severity scores are listed but should not be the primary driver for prioritisation — attack chain placement and blast radius matter more.

03 CVE-2026-0257 — Palo Alto GlobalProtect (Critical)

Type: Authentication bypass in the GlobalProtect SSL-VPN portal
Exploitation: Confirmed — weaponised within days of disclosure
Attack chain role: Initial access → direct network entry

An unauthenticated attacker can bypass the VPN authentication layer entirely, gaining access to the internal network without credentials. Because GlobalProtect is an internet-facing trust boundary, successful exploitation grants the same network position as a legitimate remote employee — before a single endpoint is touched.

Why it matters: VPNs sit at the outer trust perimeter. Credential-based controls downstream assume the perimeter held. Once bypassed, SIEM alerts based on "unusual login location" are silent.

bash 
# Check exposure -- identify GlobalProtect portals in your asset inventory
nmap -p 443 --script ssl-cert,http-title <target-range> | grep -i 'globalprotect\|palo alto'

# Shodan query for exposed instances (for authorised asset discovery)
# shodan search 'http.title:"GlobalProtect" http.component:"Palo Alto"'

# Check installed PAN-OS version via API (with credentials)
curl -k 'https://<firewall>/api/?type=version'

# CISA KEV check -- pull current Known Exploited Vulnerabilities catalogue
curl -s https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json \
  | python3 -c "import json,sys; [print(e['cveID'],e['vendorProject']) for e in json.load(sys.stdin)['vulnerabilities'] if 'Palo' in e.get('vendorProject','')]"

04 CVE-2026-32201 — Microsoft SharePoint (High)

Type: Remote code execution via malicious page/web part deserialization
Exploitation: Confirmed — observed in targeted intrusions against enterprise environments
Attack chain role: Initial access → internal reconnaissance → lateral movement

SharePoint is not just a file server — it is the organisational memory of most enterprises: org charts, project documents, finance workflows, HR records, IT run-books. Exploitation grants an attacker authenticated read/write access to this intelligence before any EDR on a workstation fires.

Why it matters: Post-exploitation data harvesting from SharePoint is virtually indistinguishable from legitimate user activity at the network level. Exfiltration blends into normal SharePoint traffic.

bash 
# Identify exposed SharePoint instances
nmap -p 443 --script http-title,http-headers <target> | grep -i sharepoint

# Check SharePoint version (on-prem, authenticated)
curl -s -k 'https://<sharepoint>/_vti_pvt/service.cnf' | grep vti_extenderversion

# Hunt for evidence of exploitation in SharePoint ULS logs
# On SharePoint server:
Get-SPLogEvent -StartTime (Get-Date).AddDays(-7) | Where-Object {$_.Message -match 'deserialization|exception|error'}

# Check for suspicious web parts added recently
Get-SPWeb -Limit all | Get-SPWebPart | Where-Object {$_.Created -gt (Get-Date).AddDays(-14)}

05 CVE-2026-33825 & CVE-2026-45498 — Microsoft Defender (High)

Type: Two separate vulnerabilities — privilege escalation (33825) and security disruption / tamper protection bypass (45498)
Exploitation: Both confirmed in-the-wild
Attack chain role: Privilege escalation, defense evasion, visibility reduction

These are chained in observed intrusions: 33825 escalates a low-privilege process to SYSTEM, then 45498 disables Defender's tamper protection and real-time monitoring without triggering standard alerts. The result is a machine that appears protected but is not.

This is a two-stage blind: the attacker first gains elevation, then silences the only tool monitoring the escalation.

powershell 
# Check if Defender tamper protection is still enabled
Get-MpComputerStatus | Select-Object -Property IsTamperProtected, RealTimeProtectionEnabled, AMServiceEnabled

# Check Defender service state and last update
Get-Service -Name WinDefend | Select-Object Status, StartType
Get-MpComputerStatus | Select-Object AntivirusSignatureLastUpdated

# Hunt for tamper protection changes in Windows Event Log
Get-WinEvent -LogName 'Microsoft-Windows-Windows Defender/Operational' |
  Where-Object {$_.Id -in @(5001,5007,5010,5012)} |
  Select-Object TimeCreated, Id, Message

# Event IDs of interest:
# 5001 - Real-time protection disabled
# 5007 - Configuration changed
# 5010 - Antimalware disabled
# 5012 - On-access scanning disabled

06 CVE-2026-6973 — Ivanti EPMM (Critical)

Type: Unauthenticated remote code execution in Ivanti Endpoint Manager Mobile (formerly MobileIron)
Exploitation: Confirmed — nation-state and ransomware actors both observed
Attack chain role: Enterprise access → device control → policy manipulation

EPMM manages every enrolled mobile device in an organisation: push configurations, enforce policies, wipe devices, deploy apps, and access corporate email. Exploitation gives an attacker the ability to push malicious configuration profiles to thousands of managed devices silently.

Why it matters: A compromised MDM is a force-multiplier. It turns one server breach into fleet-wide compromise of every enrolled phone, tablet, and laptop.

bash 
# Identify exposed Ivanti EPMM / MobileIron instances
curl -sk https://<target>/mifs/user/login.jsp -I | grep -i 'server\|x-powered-by\|mobileiron'

# Check version from API endpoint (no auth required on unpatched)
curl -sk 'https://<target>/api/v1/metadata/version'

# Shodan dork (authorised recon)
# shodan search 'http.favicon.hash:<ivanti_hash> country:US'

# Check EPMM access logs for exploitation indicators
# Look for unauthenticated API calls to sensitive endpoints:
grep -E 'POST.*/api/v1/(user|device|config).*401|403' /var/log/mi*.log | tail -50

07 Macro Trends

1. Patch-to-Exploit Compression
The window between public CVE disclosure and mass exploitation is shrinking measurably. Several Q2 2026 CVEs saw weaponised PoCs within 48–72 hours of NVD publication. Patch SLAs of 30 days are no longer adequate for critical internet-facing systems — target 24–48 hours for CVSS ≥9 on perimeter assets.

2. Security Tools as Attack Surface
Defender (twice), CrowdStrike, SentinelOne, and Tenable products all had CVEs with confirmed exploitation in the past 12 months. The implicit assumption that security products are hardened and trusted deserves re-examination. Apply vendor patches for security software first, not last.

3. Identity-First Intrusions
Malware deployment is frequently the last step, not the first. Observed kill-chains increasingly show: VPN bypass → credential harvest → lateral movement via legitimate auth → data exfil → ransomware only if prior objectives failed. Standard AV/EDR detects none of the first four phases.

4. AI-Assisted Weaponisation
PoC-to-production exploit timelines are compressing in part because LLMs lower the barrier to patch diffing and variant generation. Expect this trend to accelerate.

09 Defender Priorities

Based on Q2 2026 exploitation patterns, the highest-impact defensive actions in order:

  1. Patch internet-facing auth/VPN first — GlobalProtect, Pulse, Fortinet, Cisco AnyConnect. These are where intrusions start. 24-hour patch SLA.
  2. Audit MDM platform access — Who can call the Ivanti/JAMF/Intune API? Enable MFA on admin consoles. Log all device config push events.
  3. Monitor security tool health — Alert on Defender tamper protection changes. Verify EDR agent status from the management console, not from the endpoint itself.
  4. SharePoint anomaly detection — Baseline normal access patterns; alert on bulk document downloads, off-hours access, and new web part deployments.
  5. Credential hygiene — Identity-first intrusions succeed because credentials are reused or weakly protected. Enforce phishing-resistant MFA (FIDO2/passkeys) on all admin accounts.
  6. Threat hunt retroactively — Search logs for known C2 IPs/domains from the past 90 days. Attackers may have been present long before detection.

Key Takeaways:
  • The biggest Q2 2026 risk is not malware — it is attackers moving through your network using legitimate trust mechanisms after bypassing the perimeter
  • CVSS scores alone do not reflect operational risk; attack chain placement (initial access vs. post-exploitation) changes everything
  • If you can only do one thing: pull the CISA KEV feed weekly and treat every entry as actively exploited against something you own
