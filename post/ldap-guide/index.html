LDAP – Directory Enumeration & Active Directory Recon

2026-06-03 · 1 min read · Tools · ldap active-directory enumeration pentesting

LDAP is the backbone of Active Directory. Learn to query it anonymously and with credentials to extract users, groups, computers, and policies — essential for both AD administration and offensive enumeration.

01 What is LDAP?

Lightweight Directory Access Protocol (LDAP) is a protocol for reading and writing directory services — most commonly Microsoft Active Directory. It organises objects (users, groups, computers, OUs) in a hierarchical tree.

PortProtocolNotes
389LDAP (plaintext)Standard queries
636LDAPS (TLS)Encrypted
3268Global CatalogForest-wide searches
3269GC over TLSEncrypted GC

02 LDAP Structure

Every object has a Distinguished Name (DN) that describes its full path:

text 
CN=John Smith,OU=Sales,DC=corp,DC=local
ComponentMeaning
`DC`Domain Component (`corp.local` → `DC=corp,DC=local`)
`OU`Organisational Unit (like a folder)
`CN`Common Name (object name)
`DN`Full distinguished name (path to object)
`RDN`Relative DN (just the final component)

03 ldapsearch – The Core Tool

ldapsearch queries an LDAP server directly from the command line. Syntax: ldapsearch [options] [filter] [attributes]

bash 
# Anonymous bind -- check if null session is allowed
ldapsearch -x -H ldap://192.168.1.10 -b '' -s base '(objectclass=*)' namingContexts

# Get base DN of the domain
ldapsearch -x -H ldap://192.168.1.10 -b '' -s base namingContexts

# Authenticated query (plaintext credentials)
ldapsearch -x -H ldap://192.168.1.10 -D 'CN=jsmith,DC=corp,DC=local' -w 'Password123' \
  -b 'DC=corp,DC=local' '(objectClass=user)'

# Shorthand with UPN-style bind DN
ldapsearch -x -H ldap://192.168.1.10 -D 'jsmith@corp.local' -w 'Password123' \
  -b 'DC=corp,DC=local' '(objectClass=user)'

04 Anonymous / Null Bind Enumeration

Many older or misconfigured LDAP servers allow queries without credentials — a null bind. This is a critical misconfiguration in AD environments.

bash 
# Test for anonymous bind
ldapsearch -x -H ldap://192.168.1.10 -b 'DC=corp,DC=local'

# Dump all users anonymously
ldapsearch -x -H ldap://192.168.1.10 -b 'DC=corp,DC=local' '(objectClass=person)' \
  cn sAMAccountName mail userPrincipalName

# Dump all groups
ldapsearch -x -H ldap://192.168.1.10 -b 'DC=corp,DC=local' '(objectClass=group)' \
  cn member description

# Dump all computers
ldapsearch -x -H ldap://192.168.1.10 -b 'DC=corp,DC=local' '(objectClass=computer)' \
  cn dNSHostName operatingSystem

05 Useful LDAP Filters

FilterReturns
`(objectClass=user)`All user objects
`(objectClass=computer)`All computer objects
`(objectClass=group)`All groups
`(objectClass=organizationalUnit)`All OUs
`(objectCategory=person)`People (users + contacts)
`(memberOf=CN=Domain Admins,CN=Users,DC=corp,DC=local)`Members of Domain Admins
`(userAccountControl:1.2.840.113556.1.4.803:=2)`Disabled accounts
`(userAccountControl:1.2.840.113556.1.4.803:=65536)`Password never expires
`(badPwdCount>=1)`Accounts with failed logins
`(adminCount=1)`Privileged / protected accounts
`(servicePrincipalName=*)`Kerberoastable accounts
bash 
# Find Domain Admins members
ldapsearch -x -H ldap://dc01 -D 'user@corp.local' -w 'pass' \
  -b 'DC=corp,DC=local' \
  '(memberOf=CN=Domain Admins,CN=Users,DC=corp,DC=local)' \
  cn sAMAccountName

# Find Kerberoastable accounts (have SPN set)
ldapsearch -x -H ldap://dc01 -D 'user@corp.local' -w 'pass' \
  -b 'DC=corp,DC=local' \
  '(&(objectClass=user)(servicePrincipalName=*))' \
  cn sAMAccountName servicePrincipalName

# Find accounts with password never expires
ldapsearch -x -H ldap://dc01 -D 'user@corp.local' -w 'pass' \
  -b 'DC=corp,DC=local' \
  '(userAccountControl:1.2.840.113556.1.4.803:=65536)' \
  cn sAMAccountName

# Find description fields (often contain passwords)
ldapsearch -x -H ldap://dc01 -D 'user@corp.local' -w 'pass' \
  -b 'DC=corp,DC=local' \
  '(&(objectClass=user)(description=*))' \
  cn description sAMAccountName

06 ldapdomaindump – Structured AD Dump

ldapdomaindump extracts the entire domain structure and outputs HTML/JSON/CSV reports — ideal for a quick overview.

bash 
# Install
pip3 install ldapdomaindump

# Dump with credentials
ldapdomaindump -u 'corp.local\jsmith' -p 'Password123' 192.168.1.10

# Dump with UPN format
ldapdomaindump -u 'jsmith@corp.local' -p 'Password123' ldap://192.168.1.10 -o ./ldap-dump/

# Anonymous (if null bind allowed)
ldapdomaindump 192.168.1.10 -o ./ldap-anon/

# Output files created:
# domain_users.html / .json / .grep
# domain_groups.html / .json / .grep
# domain_computers.html / .json / .grep
# domain_policy.html / .json / .grep

07 windapsearch – Fast AD Enumeration

windapsearch is a Go tool built specifically for AD LDAP enumeration with pre-built query modules.

bash 
# Install
git clone https://github.com/ropnop/windapsearch && cd windapsearch
go build -o windapsearch cmd/windapsearch/main.go

# Enumerate all users
./windapsearch -d corp.local -u jsmith@corp.local -p 'Password123' -m users

# Enumerate Domain Admins
./windapsearch -d corp.local -u jsmith@corp.local -p 'Password123' -m privileged-users

# Enumerate computers
./windapsearch -d corp.local -u jsmith@corp.local -p 'Password123' -m computers

# Find unconstrained delegation accounts
./windapsearch -d corp.local -u jsmith@corp.local -p 'Password123' -m unconstrained

# Custom LDAP filter
./windapsearch -d corp.local -u jsmith@corp.local -p 'Password123' \
  --filter '(adminCount=1)' --attrs cn,sAMAccountName

08 Nmap NSE Scripts

bash 
# Check if anonymous bind is allowed
nmap -p 389 --script ldap-rootdse 192.168.1.10

# Brute-force credentials
nmap -p 389 --script ldap-brute --script-args ldap.base='DC=corp,DC=local' 192.168.1.10

# Search with credentials
nmap -p 389 --script ldap-search \
  --script-args ldap.username='CN=jsmith,DC=corp,DC=local',ldap.password='Password123',\
ldap.qfilter='users',ldap.attrib='sAMAccountName' 192.168.1.10

09 Python ldap3 – Scripted Queries

python 
from ldap3 import Server, Connection, ALL, NTLM

server = Server('192.168.1.10', get_info=ALL)

# Bind with credentials
conn = Connection(server, user='CORP\\jsmith', password='Password123', authentication=NTLM)
conn.bind()

# Search for all users
conn.search(
    search_base='DC=corp,DC=local',
    search_filter='(objectClass=user)',
    attributes=['cn', 'sAMAccountName', 'mail', 'description']
)

for entry in conn.entries:
    print(entry.sAMAccountName, entry.description)

# Anonymous bind attempt
conn_anon = Connection(server)
print('Anonymous bind:', conn_anon.bind())

10 Real-World Use Cases

IT Administration / Identity Management
- Query AD to generate user reports, audit group memberships, or find stale accounts for cleanup
- Automate user provisioning scripts using ldap3 — create, modify, and disable accounts programmatically
- Find all accounts with passwords set to never expire for a security compliance audit
- Integrate applications with AD authentication using LDAP bind for SSO

Penetration Testing / Red Team
- After gaining a domain foothold, enumerate users and groups to identify high-value targets
- Find service accounts with SPNs set — prime targets for Kerberoasting
- Check description fields — admins frequently store temporary passwords there
- Map group memberships to understand privilege paths: who can become Domain Admin?
- Find computers with unconstrained delegation — a classic privilege escalation route
- Dump the entire domain to JSON with ldapdomaindump and analyse offline

11 CTF / HTB Workflow

bash 
# Step 1: Discover LDAP
nmap -p 389,636,3268 --open <target>

# Step 2: Get base DN (works without credentials)
ldapsearch -x -H ldap://<target> -b '' -s base namingContexts

# Step 3: Try anonymous bind
ldapsearch -x -H ldap://<target> -b 'DC=corp,DC=local' '(objectClass=*)' | head -50

# Step 4: With creds -- dump everything
ldapdomaindump -u 'corp.local\<user>' -p '<pass>' <target> -o ./dump/

# Step 5: Hunt for credentials in description fields
ldapsearch -x -H ldap://<target> -D '<user>@corp.local' -w '<pass>' \
  -b 'DC=corp,DC=local' '(description=*)' cn sAMAccountName description \
  | grep -i 'description\|cn:'

# Step 6: Find SPNs for Kerberoasting
ldapsearch -x -H ldap://<target> -D '<user>@corp.local' -w '<pass>' \
  -b 'DC=corp,DC=local' '(servicePrincipalName=*)' sAMAccountName servicePrincipalName

# Step 7: Find AS-REP roastable users (no preauth required)
ldapsearch -x -H ldap://<target> -D '<user>@corp.local' -w '<pass>' \
  -b 'DC=corp,DC=local' \
  '(userAccountControl:1.2.840.113556.1.4.803:=4194304)' \
  sAMAccountName
CTF Tips:
  • Anonymous bind on port 389 is a common first-blood path on HTB Windows machines
  • The description attribute is a goldmine — filter with (description=*) and grep for pass, temp, cred
  • adminCount=1 finds accounts that are (or were) in privileged groups
  • If you have any valid domain creds, ldapdomaindump gives you a full HTML report in seconds
  • LDAP port 389 open but 636 closed? Try plaintext — credentials in the clear may be captured by other challenges on the network

12 Quick Reference

Key ldapsearch Flags:
  • -x — simple authentication (not SASL)
  • -H ldap://<host> — server URI
  • -D '<bind DN>' — bind as this identity
  • -w '<password>' — password (use -W to prompt)
  • -b '<base DN>' — where to start searching
  • -s base|one|sub — search scope
  • -L / -LL / -LLL — progressively cleaner output

Default Ports: 389 (LDAP), 636 (LDAPS), 3268 (Global Catalog)

One-liner user dump:
ldapsearch -x -H ldap://<dc> -D 'user@corp.local' -w 'pass' -b 'DC=corp,DC=local' '(objectClass=user)' sAMAccountName | grep sAMAccountName
