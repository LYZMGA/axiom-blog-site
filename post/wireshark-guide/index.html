Wireshark — Network Traffic Analysis

2026-06-04 · 2 min read · Security · wireshark network pcap traffic analysis pentesting

Complete guide to Wireshark — capture filters, display filters, protocol dissection, TLS decryption, and real-world traffic analysis workflows for security research and CTFs.

01 What Is Wireshark?

Wireshark is the world's most widely used network protocol analyser. It captures live packets from any interface and lets you inspect every byte of every frame — from Ethernet headers down to application-layer payloads. Whether you're debugging a protocol, hunting for credentials in cleartext, or reconstructing a compromised host's activity from a PCAP, Wireshark is the baseline tool.

Use caseExample
Protocol debuggingWhy is my TLS handshake failing?
Credential huntingHTTP Basic Auth, FTP, Telnet in cleartext
Malware analysisC2 beacon intervals, DNS tunnelling
CTF / challengeExtract files from PCAP, decode custom protocols
Network reconMap hosts, services, OS fingerprints from traffic

02 Installation

bash 
# Kali / Debian
apt install wireshark

# Allow non-root capture (adds your user to wireshark group)
sudo dpkg-reconfigure wireshark-common   # choose Yes
sudo usermod -aG wireshark $USER
newgrp wireshark

# CLI-only (no GUI needed)
apt install tshark

# macOS
brew install --cask wireshark

On Linux, raw socket capture requires CAP_NET_RAW. Adding yourself to the wireshark group grants this without running as root.

03 Capture Filters

Capture filters use BPF (Berkeley Packet Filter) syntax and are applied before packets hit the buffer — essential for high-traffic environments.

bash 
# By host
host 192.168.1.10
not host 192.168.1.1          # exclude default gateway noise

# By port
port 443
port 80 or port 443
not port 22                    # suppress SSH

# By protocol
icmp
udp
tcp

# By direction
src host 10.0.0.5
dst net 10.0.0.0/24

# Combinations
tcp and host 10.10.10.5 and port 80
net 192.168.0.0/16 and not broadcast

# Capture only SYN packets (portscans, new connections)
tcp[tcpflags] & tcp-syn != 0 and tcp[tcpflags] & tcp-ack == 0

Set capture filters in: Capture → Options → Enter a capture filter

04 Display Filters

Display filters run on already-captured packets and use Wireshark's own rich filter language. Far more powerful than BPF.

bash 
# Protocol
http
dns
smb
ftp
telnet
ssl or tls

# IP address
ip.addr == 10.10.10.5
ip.src == 192.168.1.0/24
ip.dst != 10.0.0.1

# Port
tcp.port == 4444
udp.port == 53

# HTTP specifics
http.request.method == "POST"
http.request.uri contains "login"
http.response.code == 200
http.cookie contains "session"

# DNS
dns.qry.name contains "evil"
dns.flags.rcode != 0          # DNS errors

# TCP flags
tcp.flags.syn == 1 and tcp.flags.ack == 0   # SYN scan
tcp.flags.rst == 1                           # resets

# Credentials in cleartext
ftp.request.command == "PASS"
telnet                                        # entire stream is cleartext
http.authbasic                                # HTTP Basic Auth header

# Large transfers
frame.len > 1400

# Specific string in payload
tcp contains "password"
frame contains "flag{"

05 Following Streams

The single most useful feature for application-layer analysis.

text 
Right-click any packet → Follow → TCP Stream  (or UDP / TLS / HTTP)

This reassembles the full conversation in both directions, colour-coded — client in red, server in blue. You can:
- Read cleartext HTTP requests and responses in full
- See FTP commands and directory listings
- Extract the raw bytes of a file transfer
- Copy as hex, C array, or raw binary

In the Follow Stream dialog — use the Show and save data as dropdown to switch between ASCII, hex, raw, and base64 views.

06 Exporting Objects (File Extraction)

Wireshark can reconstruct transferred files directly from traffic.

text 
File → Export Objects → HTTP
File → Export Objects → SMB
File → Export Objects → DICOM / IMF / TFTP

This lists every file transferred over the selected protocol. Click Save All to dump them all to a directory. Works on:
- HTTP downloads (EXE, ZIP, PDF, images)
- SMB file shares
- TFTP firmware uploads

CTF tip: A PCAP with image files hidden in HTTP traffic is a classic challenge. Export objects, then run file * to identify them regardless of extension.

07 TLS Decryption

Method 1 — Pre-master secret log (browsers / apps)

Set the environment variable before launching the browser:

bash 
export SSLKEYLOGFILE=~/tls-keys.log
chromium &
# or
firefox &

Then in Wireshark:

text 
Edit → Preferences → Protocols → TLS
(Pre)-Master-Secret log filename → ~/tls-keys.log

All TLS traffic from that browser session is now decryptable in real time.

Method 2 — Server private key

If you have the server's RSA private key (CTF, test lab, your own server):

text 
Edit → Preferences → Protocols → TLS → RSA Keys list → Add
IP: <server IP>   Port: 443   Protocol: http   Key file: server.key

This only works for RSA key exchange — not ECDHE (forward-secret) ciphers. Most modern servers use ECDHE, so the SSLKEYLOGFILE method is more universally useful.

09 tshark — CLI Wireshark

tshark is the headless version — essential for scripting, remote capture, and large PCAP processing.

bash 
# Live capture to file
tshark -i eth0 -w capture.pcap

# Read a PCAP
tshark -r capture.pcap

# Apply display filter
tshark -r capture.pcap -Y 'http.request.method == "POST"'

# Extract specific fields
tshark -r capture.pcap -Y dns -T fields -e ip.src -e dns.qry.name

# Extract HTTP host + URI
tshark -r capture.pcap -Y http.request \
  -T fields -e http.host -e http.request.uri

# Follow TCP stream 0
tshark -r capture.pcap -q -z follow,tcp,ascii,0

# Count conversations
tshark -r capture.pcap -q -z conv,ip

# Decrypt TLS with key log
tshark -r capture.pcap -o tls.keylog_file:tls-keys.log -Y http

# Extract credentials from FTP
tshark -r capture.pcap -Y ftp -T fields \
  -e ip.src -e ftp.request.command -e ftp.request.arg

10 Real-World Pentest Workflow

On-path / tap capture

bash 
# Capture on interface facing the target network
tshark -i eth0 -w pentest-$(date +%Y%m%d).pcap

# Quick wins — look for cleartext auth immediately
tshark -r pentest.pcap -Y 'ftp or telnet or http.authbasic' \
  -T fields -e ip.src -e ip.dst -e text

After capture — triage steps
1. Protocol hierarchy → identify unexpected protocols
2. Conversations → find the noisiest or most suspicious IPs
3. Filter dns → look for unusual domain resolutions
4. Filter http → follow streams on interesting POST requests
5. Export objects → recover any transferred files
6. Filter tcp.flags.syn==1 && tcp.flags.ack==0 → map port scan activity

Credential extraction one-liner

bash 
tshark -r capture.pcap -Y 'http.request.method=="POST"' \
  -T fields -e ip.src -e http.host -e http.request.uri \
  -e http.file_data 2>/dev/null | grep -i 'pass\|user\|login\|token'

11 CTF Workflow

Most CTF PCAP challenges fall into one of these patterns:

bash 
# 1. String hunt — flag in cleartext traffic
strings capture.pcap | grep -i 'flag{\|CTF{\|htb{'
tshark -r capture.pcap -Y 'frame contains "flag{"'

# 2. File extraction
# File → Export Objects → HTTP (or SMB)
# Then: file * && binwalk *

# 3. Credential leak
tshark -r capture.pcap -Y 'http.authbasic or ftp.request.command=="PASS"'
# Base64 decode HTTP Basic: echo 'dXNlcjpwYXNz' | base64 -d

# 4. Custom/unknown protocol
# Statistics → Protocol Hierarchy → find the odd one out
# Follow TCP stream → look at the raw bytes
# Use Data field: data contains <hex bytes>

# 5. DNS exfiltration
tshark -r capture.pcap -Y dns -T fields -e dns.qry.name \
  | sort | uniq | grep -v 'google\|microsoft\|apple'

# 6. Decrypting TLS (if key provided)
tshark -r capture.pcap -o tls.keylog_file:provided.log -Y http2
