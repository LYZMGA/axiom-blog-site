SNMP – Network Intelligence & Enumeration

2026-06-03 · 1 min read · Tools · snmp enumeration network pentesting

SNMP exposes a wealth of device info — system details, interfaces, running processes, installed software. Learn to query it, brute-force community strings, and extract data for both ops and offensive use.

01 What is SNMP?

Simple Network Management Protocol (SNMP) is a UDP-based protocol for monitoring and managing network devices — routers, switches, servers, printers, IoT hardware. It runs on port 161 (queries) and port 162 (traps/notifications).

Devices expose a MIB (Management Information Base) — a tree of OIDs (Object Identifiers) that map to readable values like hostname, uptime, interfaces, routes, running processes, and installed software.

02 SNMP Versions

VersionAuthNotes
v1Community string (plaintext)Oldest, no encryption
v2cCommunity string (plaintext)Faster bulk queries, still cleartext
v3Username + password + encryptionSecure, but complex to configure
Warning

v1 and v2c are still extremely common. Community strings (public, private) act as passwords and are transmitted in cleartext — anyone on the network can capture them with Wireshark.

03 Tool Overview

ToolPurpose
`snmpwalk`Walk entire MIB subtree
`snmpget`Query a specific OID
`snmpset`Write a value (if allowed)
`onesixtyone`Fast community string brute-force
`snmp-check`Formatted enumeration dump
`nmap`NSE SNMP scripts
`snmpenum` / `snmpbulkwalk`Faster bulk enumeration

04 snmpwalk – Walk the MIB

snmpwalk queries all OIDs under a given subtree recursively.

bash 
# Walk everything (v2c, community string 'public')
snmpwalk -v2c -c public 192.168.1.1

# Walk a specific OID subtree
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.1    # system info
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.25.4 # running processes
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.25.6 # installed software
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.4.1.77   # Windows user accounts

# Use MIB names instead of numeric OIDs
snmpwalk -v2c -c public -m all 192.168.1.1 sysDescr

# SNMPv3 with auth + encryption
snmpwalk -v3 -l authPriv -u admin -a SHA -A 'authpass' -x AES -X 'privpass' 192.168.1.1

05 Key OIDs to Know

OIDDescription
`1.3.6.1.2.1.1.1`sysDescr — OS, version, hardware
`1.3.6.1.2.1.1.5`sysName — hostname
`1.3.6.1.2.1.2.2`ifTable — network interfaces
`1.3.6.1.2.1.4.20`ipAddrTable — IP addresses
`1.3.6.1.2.1.4.21`ipRouteTable — routing table
`1.3.6.1.2.1.25.1`hrSystem — system uptime, users
`1.3.6.1.2.1.25.4.2`hrSWRunName — running processes
`1.3.6.1.2.1.25.6.3`hrSWInstalledName — installed software
`1.3.6.1.4.1.77.1.2.25`Windows user accounts (Microsoft MIB)

06 snmpget – Single OID Query

bash 
# Get system description
snmpget -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1.0

# Get hostname
snmpget -v2c -c public 192.168.1.1 sysName.0

# Get uptime
snmpget -v2c -c public 192.168.1.1 sysUpTimeInstance

07 onesixtyone – Community String Brute-Force

When the community string is unknown, onesixtyone tries a wordlist rapidly.

bash 
# Single target with a wordlist
onesixtyone -c /usr/share/seclists/Discovery/SNMP/common-snmp-community-strings.txt 192.168.1.1

# Scan a subnet
onesixtyone -c community.txt -i hosts.txt

# Quick test of common defaults
onesixtyone 192.168.1.1 public private manager

# Using rockyou-style list
cat /usr/share/wordlists/rockyou.txt | head -1000 > /tmp/snmp-words.txt
onesixtyone -c /tmp/snmp-words.txt 192.168.1.1

08 snmp-check – Formatted Enumeration

snmp-check organizes output into readable sections: system info, network, processes, storage, software.

bash 
# Basic enumeration
snmp-check 192.168.1.1

# Specify version and community string
snmp-check -v 2c -c private 192.168.1.1

# Verbose output (more detail)
snmp-check -v 2c -c public -d 192.168.1.1

# Save to file
snmp-check 192.168.1.1 | tee snmp-enum.txt

09 Nmap NSE Scripts

bash 
# Discover SNMP and check community strings
nmap -sU -p 161 --script snmp-brute 192.168.1.1

# Full info dump with known community string
nmap -sU -p 161 --script snmp-info 192.168.1.1

# Enumerate network interfaces
nmap -sU -p 161 --script snmp-interfaces 192.168.1.1

# Get running processes
nmap -sU -p 161 --script snmp-processes 192.168.1.1

# List open TCP ports on target via SNMP
nmap -sU -p 161 --script snmp-netstat 192.168.1.1

# Run all SNMP scripts at once
nmap -sU -p 161 --script snmp-* 192.168.1.1

# SNMP UDP discovery across subnet
nmap -sU -p 161 --open 192.168.1.0/24

10 snmpbulkwalk – Faster Enumeration

bash 
# Bulk walk is faster than snmpwalk for large MIBs
snmpbulkwalk -v2c -c public -Cn0 -Cr10 192.168.1.1

# Walk specific subtree fast
snmpbulkwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.25

# Output only values (no OID names)
snmpbulkwalk -v2c -c public -Ov 192.168.1.1 sysDescr

11 snmpset – Write Values

If the community string allows writes (usually private), you can modify device configuration remotely.

bash 
# Change system contact (string type 's')
snmpset -v2c -c private 192.168.1.1 sysContact.0 s 'admin@corp.com'

# Change hostname
snmpset -v2c -c private 192.168.1.1 sysName.0 s 'new-hostname'

# Type codes: i=integer, s=string, x=hex, o=OID, t=timeticks
Warning

Write access via SNMP can allow device reconfiguration, interface shutdown, or firmware changes on managed switches and routers. Always confirm authorization before using snmpset in production environments.

12 Extracting Useful Data

bash 
# Pull running processes and filter for interesting ones
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.25.4.2.1.2 | grep -iE 'sql|ftp|ssh|web|apache|nginx|tomcat'

# List all installed software versions
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.25.6.3.1.2

# Dump Windows users (Microsoft MIB)
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.4.1.77.1.2.25

# Get all IP addresses configured on device
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.4.20.1.1

# Routing table
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.4.21

13 Real-World Use Cases

Network Operations / Monitoring
- Poll switch/router uptime, interface errors, and bandwidth via SNMP to feed into Grafana / Prometheus / PRTG
- Discover all devices on a subnet and map their hostnames and software versions for an asset inventory
- Monitor CPU/memory on servers without installing agents — many hypervisors and NAS devices expose health via SNMP
- Receive SNMP traps (port 162) for real-time alerts: link down, high temperature, disk failure

Penetration Testing / Internal Assessment
- After finding open UDP 161, walk the MIB to enumerate OS version, hostname, running services, and user accounts without any credentials
- Extract routing tables to map internal network segments that aren't directly reachable from your foothold
- Find software versions in the installed software list and cross-reference with CVE databases
- Discover credentials or sensitive data exposed in custom MIB extensions (sysDescr fields sometimes contain config snippets)

14 CTF / HTB Workflow

bash 
# Step 1: Confirm SNMP is open
nmap -sU -p 161 --open <target>

# Step 2: Try default community strings
onesixtyone -c /usr/share/seclists/Discovery/SNMP/common-snmp-community-strings.txt <target>

# Step 3: Full walk with discovered community string
snmpwalk -v2c -c <community> <target> > snmp-full.txt

# Step 4: Hunt for juicy data
grep -iE 'pass|key|secret|token|user|admin|flag' snmp-full.txt

# Step 5: Windows-specific — users and shares
snmpwalk -v2c -c <community> <target> 1.3.6.1.4.1.77.1.2.25    # users
snmpwalk -v2c -c <community> <target> 1.3.6.1.4.1.77.1.2.27    # shares

# Step 6: Check if write access is possible
snmpset -v2c -c private <target> sysName.0 s 'test' 2>&1
CTF Tips:
  • Community strings found in CTFs are often machine names, challenge themes, or simple dictionary words — try onesixtyone with SecLists first
  • Windows boxes: the Microsoft MIB (1.3.6.1.4.1.77) often leaks local user accounts and group memberships
  • Check sysDescr and sysContact fields — admins sometimes store notes, emails, or version info there
  • If SNMP v3 is used, nmap snmp-brute can try credential pairs from a wordlist

15 Quick Reference

Info

Common Community Strings: public, private, manager, community, cisco, router, switch

Key Flags:
- -v1 / -v2c / -v3 — SNMP version
- -c <string> — community string
- -m all — load all MIBs (use named OIDs)
- -Ov — output values only
- -t 1 -r 0 — reduce timeout/retries for fast scanning

Ports: UDP 161 (polling), UDP 162 (traps)

One-liner full enum:
snmp-check -v 2c -c public <target> | tee snmp.txt && grep -iE 'user|pass|admin' snmp.txt
