SMTP — Field Guide

2026-06-03 · 1 min read · Tools · smtp email enumeration tools network

Banner grabbing, user enumeration via VRFY/RCPT, open relay testing with swaks, and a CTF workflow for turning an open relay into code execution.

01 What is SMTP

SMTP (Simple Mail Transfer Protocol) handles outbound email delivery — sending mail from client to server and between servers.

Ports:

PortUse
`25`Server-to-server relay (MTA). Blocked by most ISPs on residential IPs.
`465`SMTP over implicit TLS (legacy, but still common).
`587`Submission port — authenticated client-to-server. Standard for apps.
`2525`Alternative submission port used by some providers.

For pentesters, SMTP is interesting for three reasons: user enumeration via VRFY/RCPT TO, open relay testing (can the server forward mail for any domain?), and phishing/spoofing when found on an internal network.

02 Manual Connection

SMTP is a plain-text protocol — you can speak it directly with nc or telnet. Understanding the raw commands is essential for both debugging and exploitation.

bash Raw SMTP session via netcat 
nc -nv 10.10.10.5 25

# Server responds with banner, e.g.:
# 220 mail.example.com ESMTP Postfix

EHLO attacker.com          # introduce yourself, list supported extensions
HELO attacker.com          # older, simpler version of EHLO

# Check if a user exists
VRFY root                  # verify a username
VRFY admin@example.com

# Expand a mailing list
EXPN support

# Send an email manually
MAIL FROM:<attacker@evil.com>
RCPT TO:<victim@example.com>
DATA
Subject: Test
From: attacker@evil.com
To: victim@example.com

This is a test email.
.
QUIT

Key SMTP response codes:

CodeMeaning
`220`Service ready (banner)
`250`OK — command accepted
`251`User not local, forwarding
`252`Cannot verify but will attempt delivery
`354`Start mail input (after DATA)
`450/550`Mailbox unavailable / not found
`503`Bad sequence of commands
`535`Auth failed

03 Nmap SMTP Scripts

bash Nmap SMTP enumeration 
# Service version + banner
nmap -sV -p 25,465,587 10.10.10.5

# List supported SMTP commands (EHLO extensions)
nmap -p 25 --script smtp-commands 10.10.10.5

# Enumerate users via VRFY, EXPN, RCPT
nmap -p 25 --script smtp-enum-users 10.10.10.5
nmap -p 25 --script smtp-enum-users \
  --script-args smtp-enum-users.userlist=/usr/share/wordlists/metasploit/unix_users.txt \
  10.10.10.5

# Check for open relay
nmap -p 25 --script smtp-open-relay 10.10.10.5

# Check for NTLM info disclosure
nmap -p 25 --script smtp-ntlm-info 10.10.10.5

# Run all SMTP scripts at once
nmap -p 25 --script smtp-* 10.10.10.5

04 User Enumeration

Many SMTP servers respond differently to valid vs. invalid recipients, leaking valid usernames through VRFY, EXPN, or RCPT TO responses.

Three enumeration methods:

  • VRFY — asks the server to verify a user exists. Often disabled but worth trying.
  • EXPN — expands a mailing list into addresses. Usually disabled.
  • RCPT TO — most reliable; tests whether the server accepts delivery for that address. A 250 means valid, 550 means not found.
bash smtp-user-enum 
# Install if needed
sudo apt install smtp-user-enum

# VRFY method
smtp-user-enum -M VRFY -U /usr/share/wordlists/metasploit/unix_users.txt -t 10.10.10.5

# EXPN method
smtp-user-enum -M EXPN -U userlist.txt -t 10.10.10.5

# RCPT method (most reliable)
smtp-user-enum -M RCPT -U userlist.txt -D example.com -t 10.10.10.5

# Custom port
smtp-user-enum -M VRFY -U userlist.txt -t 10.10.10.5 -p 587

# Single user check
smtp-user-enum -M VRFY -u root -t 10.10.10.5
bash Manual VRFY / RCPT enumeration loop 
# Quick VRFY loop with netcat
for user in root admin mail www-data postmaster; do
  echo -e "VRFY $user\nQUIT" | nc -w 3 10.10.10.5 25 2>/dev/null \
    | grep -v '^220\|^221' | awk -v u=$user '{print u": "$0}'
done

# RCPT TO loop — more reliable
for user in root admin mail www-data; do
  result=$(echo -e "EHLO test\nMAIL FROM:<x@x.com>\nRCPT TO:<$user@example.com>\nQUIT" \
    | nc -w 3 10.10.10.5 25 2>/dev/null | grep '^250\|^550')
  echo "$user: $result"
done

05 swaks — Send Test Emails

swaks (Swiss Army Knife for SMTP) is the best tool for sending test emails and probing SMTP servers. It handles auth, TLS, attachments, and custom headers — all from the command line.

bash swaks examples 
# Send a basic test email
swaks --to victim@example.com --from attacker@evil.com --server 10.10.10.5

# Send with authentication
swaks --to user@example.com --from user@example.com \
  --server 10.10.10.5 --port 587 \
  --auth LOGIN --auth-user user@example.com --auth-password 'Password123'

# Send over TLS
swaks --to user@example.com --from attacker@evil.com \
  --server mail.example.com --port 587 --tls

# Custom headers and body — useful for phishing tests
swaks --to victim@example.com \
  --from 'IT Support <support@example.com>' \
  --server 10.10.10.5 \
  --header 'Subject: Password Reset Required' \
  --body 'Click here: http://attacker.com/reset'

# Send with an attachment
swaks --to victim@example.com --from attacker@evil.com \
  --server 10.10.10.5 \
  --attach /tmp/payload.pdf \
  --header 'Subject: Invoice'

# Probe open relay — try to relay to an external address
swaks --to external@gmail.com --from nobody@nobody.com --server 10.10.10.5

06 Open Relay Testing

An open relay accepts mail from anyone to anyone — no authentication required. Once the default for all SMTP servers, now a critical misconfiguration that gets servers blacklisted and abused for spam.

Test by attempting to relay from an external sender to an external recipient through the target server. If it accepts, it's an open relay.

bash Open relay detection 
# Manual test with netcat
nc -nv 10.10.10.5 25
EHLO test.com
MAIL FROM:<nobody@external.com>
RCPT TO:<victim@gmail.com>
# 250 = open relay (bad)
# 550/554 = relay denied (expected)

# Automated with nmap
nmap -p 25 --script smtp-open-relay 10.10.10.5

# Automated with swaks
swaks --to test@gmail.com --from fake@attacker.com --server 10.10.10.5 \
  --quit-after RCPT --hide-all
# Exit code 0 = accepted (open relay)

07 Sending Email with Python

Useful for automating phishing simulations, alert scripts, or CTF exploit chains that need to send email:

python Python smtplib 
import smtplib
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart

# Basic unauthenticated send (open relay / internal server)
def send_plain(smtp_host, sender, recipient, subject, body):
    msg = MIMEMultipart()
    msg['From']    = sender
    msg['To']      = recipient
    msg['Subject'] = subject
    msg.attach(MIMEText(body, 'plain'))
    with smtplib.SMTP(smtp_host, 25, timeout=10) as s:
        s.sendmail(sender, recipient, msg.as_string())

# Authenticated send (port 587 + STARTTLS)
def send_auth(host, user, password, recipient, subject, body):
    msg = MIMEText(body)
    msg['Subject'] = subject
    msg['From']    = user
    msg['To']      = recipient
    with smtplib.SMTP(host, 587) as s:
        s.ehlo()
        s.starttls()
        s.login(user, password)
        s.sendmail(user, recipient, msg.as_string())

# Example
send_plain('10.10.10.5', 'attacker@evil.com', 'admin@target.com',
           'Test', 'Internal relay confirmed.')

08 Real-World Use Cases

Verify email deliverability — debug why emails land in spam:

bash Deliverability diagnostics 
# Check MX records for the target domain
dig MX example.com +short

# Test connection to the mail server directly
swaks --to test@example.com --server $(dig MX example.com +short | sort -n | head -1 | awk '{print $2}')

# Check if your sending IP is blacklisted
dig +short $(echo '5.10.10.10' | awk -F. '{print $4"."$3"."$2"."$1}').zen.spamhaus.org

# Verify SPF record accepts your sending IP
dig TXT example.com +short | grep spf

# Test STARTTLS support
swaks --to test@example.com --server mail.example.com --port 587 --tls --quit-after EHLO

Phishing simulation — authorized red team test sending a crafted email internally:

bash Internal phishing test (authorized) 
# Spoof sender via internal open relay
swaks \
  --to employee@company.com \
  --from 'CEO <ceo@company.com>' \
  --server 10.0.0.25 \
  --header 'Subject: Urgent: Action Required' \
  --body 'Please review: http://phish.redteam.lab/reset'

# With HTML body
swaks \
  --to employee@company.com \
  --from 'IT Helpdesk <helpdesk@company.com>' \
  --server 10.0.0.25 \
  --header 'Subject: Password expiry notice' \
  --header 'Content-Type: text/html' \
  --body '<p>Your password expires today. <a href="http://phish.lab">Reset here</a>.</p>'

09 CTF & HackTheBox Workflow

SMTP on CTF boxes typically leads to one of: valid usernames to spray against SSH/FTP, an open relay to deliver a payload, or credentials found in config files that unlock the mail server.

bash HTB SMTP recon checklist 
export IP=10.10.10.5

# 1. Banner grab + supported commands
nc -nv $IP 25
nmap -p 25 --script smtp-commands $IP

# 2. Check for NTLM info disclosure (leaks hostname, domain)
nmap -p 25 --script smtp-ntlm-info $IP

# 3. User enumeration — try all three methods
smtp-user-enum -M VRFY -U /usr/share/wordlists/metasploit/unix_users.txt -t $IP
smtp-user-enum -M EXPN -U /usr/share/wordlists/metasploit/unix_users.txt -t $IP
smtp-user-enum -M RCPT -U /usr/share/wordlists/metasploit/unix_users.txt -D $(nmap -p 25 --script smtp-commands $IP | grep 'Hello' | grep -oP 'Hello \K\S+') -t $IP

# 4. Test open relay
nmap -p 25 --script smtp-open-relay $IP

# 5. If open relay — send a payload to an internal user
swaks --to admin@example.htb --from root@localhost --server $IP \
  --body 'http://10.10.14.5/shell.sh' \
  --header 'Subject: Check this out'
bash Send a payload via open relay (CTF) 
# Common scenario: web app processes emails, open relay lets you trigger it

# Deliver a malicious link or attachment to a user that clicks everything
swaks --to developer@example.htb \
  --from support@example.htb \
  --server $IP \
  --header 'Subject: New ticket assigned' \
  --body 'Review ticket: http://10.10.14.5/?$(id)'

# Or send with a file attachment (e.g. a .odt with macro, csv injection)
swaks --to developer@example.htb --from noreply@example.htb \
  --server $IP \
  --attach /tmp/payload.odt \
  --header 'Subject: Monthly report'
Quick reference

nc -nv IP 25 — always start with a raw connection to read the banner.
VRFY → EXPN → RCPT TO — try each method for user enum; RCPT is most reliable.
smtp-user-enum -M RCPT needs a domain (-D) to build full addresses.
swaks is the best tool for actually sending test emails and checking relay.
NTLM info disclosure (smtp-ntlm-info) leaks hostname and domain — useful for AD boxes.
Port 587 requires auth; port 25 is where open relays live on internal networks.
